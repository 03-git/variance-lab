Per reflex.1 — the kernel has no HTTP; the no-dependency primitive here is stdlib `urllib.request` (not the `requests` library). Code:

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        print(f"usage: {sys.argv[0]} URL", file=sys.stderr)
        sys.exit(2)

    url = sys.argv[1]
    req = urllib.request.Request(url, headers={"User-Agent": "fetch.py"})

    try:
        # urllib follows redirects by default via HTTPRedirectHandler
        with urllib.request.urlopen(req) as resp:
            status = resp.status
            headers = resp.headers
            body = resp.read(500).decode("utf-8", errors="replace")
    except urllib.error.HTTPError as e:
        # HTTPError is also a response — report it rather than crashing
        status = e.code
        headers = e.headers
        body = e.read(500).decode("utf-8", errors="replace")
    except urllib.error.URLError as e:
        print(f"request failed: {e.reason}", file=sys.stderr)
        sys.exit(1)

    print(f"Status: {status}\n")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
# save as fetch.py, then:
python3 fetch.py https://example.com

# or make it executable:
chmod +x fetch.py
./fetch.py https://httpbin.org/redirect/2    # exercises redirect-following
```

Notes:
- `urllib.request.urlopen` follows redirects automatically (up to its internal limit) — no flag needed, unlike `curl -L`.
- `final_url = resp.geturl()` gives you the post-redirect URL if you want to print it too.
- For TLS/cert problems you'll get a `URLError` with the reason; this prints it to stderr and exits non-zero.
