Per reflex.1 — the stdlib primitive here is `urllib.request` (built on the socket/SSL kernel layer); no need to reach for `requests`.

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        print(f"usage: {sys.argv[0]} URL", file=sys.stderr)
        sys.exit(2)

    url = sys.argv[1]
    req = urllib.request.Request(url, headers={"User-Agent": "fetch.py"})

    try:
        # urllib follows redirects by default via HTTPRedirectHandler
        with urllib.request.urlopen(req) as resp:
            status = resp.status
            headers = resp.headers
            body = resp.read(500).decode(resp.headers.get_content_charset() or "utf-8", "replace")
    except urllib.error.HTTPError as e:
        # HTTPError is also a response — report it rather than crashing
        status = e.code
        headers = e.headers
        body = e.read(500).decode("utf-8", "replace")
    except urllib.error.URLError as e:
        print(f"request failed: {e.reason}", file=sys.stderr)
        sys.exit(1)

    print(f"Status: {status}\n")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
# save as fetch.py, then:
python3 fetch.py https://example.com

# or make it executable:
chmod +x fetch.py
./fetch.py https://httpbin.org/redirect/2
```

Notes:
- `urllib.request.urlopen` follows 3xx redirects automatically (up to 10 hops) — no extra flag needed.
- `HTTPError` is caught separately because a 404/500 is still a real response with headers and a body worth printing.
- `resp.read(500)` reads at most 500 bytes; for multibyte UTF-8 that can be slightly fewer than 500 decoded chars, with `"replace"` guarding a split codepoint. Use `resp.read()[:500]` first if you want exactly 500 chars of fully-decoded text.
