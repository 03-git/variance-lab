Kernel primitive first: HTTP here is `urllib.request` from the stdlib (sockets underneath) — no need for the `requests` wrapper.

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        sys.exit(f"usage: {sys.argv[0]} URL")
    url = sys.argv[1]

    # urllib follows redirects by default via HTTPRedirectHandler
    req = urllib.request.Request(url, headers={"User-Agent": "curl-ish/1.0"})
    try:
        resp = urllib.request.urlopen(req)
    except urllib.error.HTTPError as e:
        resp = e  # HTTPError is also a response-like object
    except urllib.error.URLError as e:
        sys.exit(f"request failed: {e.reason}")

    body = resp.read(500).decode(resp.headers.get_content_charset() or "utf-8", "replace")

    print(f"Status: {resp.status} {resp.reason}")
    print(f"Final URL: {resp.url}")
    print("\nHeaders:")
    for k, v in resp.headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
python3 fetch.py https://example.com
```

Notes:
- Redirects are followed automatically — `resp.url` shows where you landed.
- `HTTPError` is caught so 4xx/5xx still print status + headers + body instead of crashing.
- `resp.read(500)` pulls only the first 500 bytes off the socket; decode is lossy-safe with `"replace"`.
