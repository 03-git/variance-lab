Kernel primitive note (reflex.1): there's no kernel HTTP call — it's `connect(2)`/`sendto(2)` over a socket. The stdlib wrapper closest to that without pulling a dependency is `urllib.request` (which is just `http.client` on a socket). So no `requests` needed.

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        sys.exit(f"usage: {sys.argv[0]} URL")
    url = sys.argv[1]

    req = urllib.request.Request(url, headers={"User-Agent": "curl-ish/1.0"})
    try:
        # urllib follows redirects by default (HTTPRedirectHandler)
        with urllib.request.urlopen(req) as resp:
            status = resp.status
            headers = resp.headers
            body = resp.read(500).decode("utf-8", errors="replace")
            final_url = resp.url
    except urllib.error.HTTPError as e:
        # 4xx/5xx still carry headers + body
        status = e.code
        headers = e.headers
        body = e.read(500).decode("utf-8", errors="replace")
        final_url = e.url
    except urllib.error.URLError as e:
        sys.exit(f"request failed: {e.reason}")

    print(f"Final URL: {final_url}")
    print(f"Status: {status}")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
# save as fetch.py, then:
python3 fetch.py https://example.com

# or make it executable:
chmod +x fetch.py
./fetch.py https://httpbin.org/redirect/2
```

Notes:
- `urllib.request.urlopen` follows redirects automatically; `resp.url` shows where it landed.
- `HTTPError` is caught separately because urllib raises on 4xx/5xx — but the error object still exposes `.code`, `.headers`, and the body, so you get the same output for those.
- `read(500)` pulls at most 500 bytes; `decode(errors="replace")` keeps it from crashing on binary or partial-UTF-8 responses.
