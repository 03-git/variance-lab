Here's a FastAPI implementation — clean validation, no boilerplate:

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from pydantic import BaseModel, field_validator, EmailStr
from pydantic import ValidationError
import uvicorn

app = FastAPI()

class User(BaseModel):
    name: str
    email: str
    age: int

    @field_validator("name")
    @classmethod
    def name_not_empty(cls, v):
        if not v.strip():
            raise ValueError("name must not be empty")
        return v

    @field_validator("email")
    @classmethod
    def email_contains_at(cls, v):
        if "@" not in v:
            raise ValueError("email must contain @")
        return v

    @field_validator("age")
    @classmethod
    def age_in_range(cls, v):
        if not (1 <= v <= 150):
            raise ValueError("age must be between 1 and 150")
        return v

@app.post("/users", status_code=201)
async def create_user(request: Request):
    try:
        body = await request.json()
    except Exception:
        return JSONResponse(status_code=400, content={"error": "invalid JSON"})

    try:
        user = User(**body)
    except ValidationError as e:
        errors = [{"field": err["loc"][0], "message": err["msg"]} for err in e.errors()]
        return JSONResponse(status_code=400, content={"errors": errors})

    return JSONResponse(status_code=201, content=user.model_dump())

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

**To run:**

```bash
pip install fastapi uvicorn
python app.py
```

**Test it:**

```bash
# Valid — expect 201
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "alice@example.com", "age": 30}' | jq

# Missing name — expect 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@example.com", "age": 30}' | jq

# Bad email — expect 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "notanemail", "age": 30}' | jq

# Age out of range — expect 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "alice@x.com", "age": 200}' | jq
```

**Sample responses:**

```json
// 201
{"name": "Alice", "email": "alice@example.com", "age": 30}

// 400 (bad email + age)
{"errors": [
  {"field": "email", "message": "Value error, email must contain @"},
  {"field": "age",   "message": "Value error, age must be between 1 and 150"}
]}
```

Pydantic collects all validation errors before returning, so you get every field failure in one response rather than failing fast on the first. If you'd prefer Flask or stdlib `http.server`, say so.
