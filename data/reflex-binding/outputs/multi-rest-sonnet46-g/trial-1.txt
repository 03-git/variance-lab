Here's a FastAPI implementation — clean validation via Pydantic, no boilerplate:

```python
from fastapi import FastAPI, Request
from fastapi.responses import JSONResponse
from pydantic import BaseModel, field_validator, EmailStr
from pydantic import ValidationError
import re

app = FastAPI()

class UserIn(BaseModel):
    name: str
    email: str
    age: int

    @field_validator("name")
    @classmethod
    def name_nonempty(cls, v):
        if not v.strip():
            raise ValueError("name must not be empty")
        return v

    @field_validator("email")
    @classmethod
    def email_has_at(cls, v):
        if "@" not in v:
            raise ValueError("email must contain @")
        return v

    @field_validator("age")
    @classmethod
    def age_range(cls, v):
        if not (1 <= v <= 150):
            raise ValueError("age must be between 1 and 150")
        return v


@app.post("/users", status_code=201)
async def create_user(request: Request):
    try:
        body = await request.json()
    except Exception:
        return JSONResponse(status_code=400, content={"error": "invalid JSON"})

    try:
        user = UserIn(**body)
    except ValidationError as e:
        errors = [{"field": err["loc"][0], "message": err["msg"]} for err in e.errors()]
        return JSONResponse(status_code=400, content={"errors": errors})

    return JSONResponse(status_code=201, content=user.model_dump())
```

**How to run it:**

```bash
# Install dependencies
pip install fastapi uvicorn

# Run (save above as main.py, or paste into a file)
uvicorn main:app --reload --port 8000
```

**Test it:**

```bash
# Valid request → 201
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "alice@example.com", "age": 30}' | jq

# Missing name → 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@example.com", "age": 30}' | jq

# Bad email → 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "notanemail", "age": 30}' | jq

# Age out of range → 400
curl -s -X POST http://localhost:8000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "a@b.com", "age": 200}' | jq
```

**Sample responses:**

```json
// 201
{"name": "Alice", "email": "alice@example.com", "age": 30}

// 400
{"errors": [{"field": "email", "message": "Value error, email must contain @"}]}

// 400 (multiple failures)
{"errors": [
  {"field": "email", "message": "Value error, email must contain @"},
  {"field": "age",   "message": "Value error, age must be between 1 and 150"}
]}
```

One note: the `age` field will return a 400 automatically if a non-integer is passed (e.g. `"age": "old"`) — Pydantic handles the type coercion failure before your validator runs.
