The testing primitive here is `assert` — no framework needed. Plain script:

```python
def celsius_to_fahrenheit(c):
    return c * 9 / 5 + 32


def test():
    assert celsius_to_fahrenheit(0) == 32
    assert celsius_to_fahrenheit(100) == 212
    assert celsius_to_fahrenheit(-40) == -40  # the fixed point

    # non-numeric input should raise TypeError
    try:
        celsius_to_fahrenheit("warm")
    except TypeError:
        pass
    else:
        raise AssertionError("expected TypeError on non-numeric input")

    print("all tests passed")


if __name__ == "__main__":
    test()
```

Run it:

```
python3 test_celsius.py
```

Exit code 0 and `all tests passed` means green; an `AssertionError` traceback means a failure.

---

If you'd rather use `pytest` (third-party, `pip install pytest`), the same tests:

```python
import pytest
from celsius import celsius_to_fahrenheit  # wherever the function lives


def test_freezing():
    assert celsius_to_fahrenheit(0) == 32

def test_boiling():
    assert celsius_to_fahrenheit(100) == 212

def test_fixed_point():
    assert celsius_to_fahrenheit(-40) == -40

def test_non_numeric():
    with pytest.raises(TypeError):
        celsius_to_fahrenheit("warm")
```

Run: `pytest test_celsius.py -v`

Note: the non-numeric test passes because `"warm" * 9` is a `str`/`int` op that Python rejects with `TypeError` before the `/ 5` — the function does no explicit validation. If you want a clearer error, add `if not isinstance(c, (int, float)): raise TypeError(...)` at the top.
