```python
import asyncio

async def handle_client(reader: asyncio.StreamReader, writer: asyncio.StreamWriter) -> None:
    addr = writer.get_extra_info("peername")
    print(f"connected: {addr}")
    try:
        async for line in reader:
            writer.write(line)
            await writer.drain()
    except asyncio.IncompleteReadError:
        pass
    finally:
        print(f"disconnected: {addr}")
        writer.close()
        await writer.wait_closed()

async def main() -> None:
    server = await asyncio.start_server(handle_client, "127.0.0.1", 8888)
    addr = server.sockets[0].getsockname()
    print(f"listening on {addr}")
    async with server:
        await server.serve_forever()

if __name__ == "__main__":
    asyncio.run(main())
```

**Run it:**

```bash
python echo_server.py
```

**Test with netcat (each in its own terminal):**

```bash
nc 127.0.0.1 8888
```

Type a line and press Enter — it echoes back. Open multiple `nc` sessions simultaneously to verify concurrency.

**How it works:**

- `asyncio.start_server` registers `handle_client` as a callback for each new connection; the event loop handles all clients on a single thread via cooperative multitasking.
- `async for line in reader` reads newline-delimited chunks; the coroutine yields back to the event loop while waiting for data, so other clients run.
- `writer.drain()` yields back to the event loop while the kernel flushes the write buffer, preventing backpressure buildup.
- No explicit connection limit — the OS backlog and file descriptor limit are the practical ceiling.
