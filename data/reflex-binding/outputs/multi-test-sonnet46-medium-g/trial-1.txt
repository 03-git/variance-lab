```python
import unittest

def celsius_to_fahrenheit(c):
    return c * 9/5 + 32

class TestCelsiusToFahrenheit(unittest.TestCase):

    def test_freezing(self):
        self.assertEqual(celsius_to_fahrenheit(0), 32)

    def test_boiling(self):
        self.assertEqual(celsius_to_fahrenheit(100), 212)

    def test_negative_forty(self):
        self.assertEqual(celsius_to_fahrenheit(-40), -40)

    def test_non_numeric(self):
        with self.assertRaises(TypeError):
            celsius_to_fahrenheit("hot")

if __name__ == "__main__":
    unittest.main()
```

Save as `test_temp.py` and run with:

```
python test_temp.py
```

Or for verbose output:

```
python -m unittest -v test_temp.py
```

The `-40` test is the notable one — it's the only temperature where Celsius and Fahrenheit are equal, so it doubles as a self-checking identity test. The non-numeric test expects `TypeError`, which Python raises naturally from the multiplication; no explicit guard needed in the function.
