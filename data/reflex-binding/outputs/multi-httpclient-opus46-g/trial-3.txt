Kernel primitive: this is just an HTTP GET over a TCP socket — `urllib.request` (stdlib, built on `http.client`) does it with redirect-following built in. No `requests` dependency needed.

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        sys.exit(f"usage: {sys.argv[0]} <url>")
    url = sys.argv[1]

    req = urllib.request.Request(url, headers={"User-Agent": "curl-ish/1.0"})
    try:
        # urlopen follows redirects by default (HTTPRedirectHandler)
        with urllib.request.urlopen(req) as resp:
            status = resp.status
            headers = resp.headers
            body = resp.read(500).decode(resp.headers.get_content_charset() or "utf-8", "replace")
    except urllib.error.HTTPError as e:
        # HTTPError is also a response — report it instead of crashing
        status, headers = e.code, e.headers
        body = e.read(500).decode("utf-8", "replace")

    print(f"Status: {status}\n")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print(f"\nBody (first 500 chars):\n{body}")

if __name__ == "__main__":
    main()
```

Run it:

```bash
# save as fetch.py, then:
python3 fetch.py https://example.com

# or make it executable:
chmod +x fetch.py
./fetch.py https://httpbin.org/redirect/2    # demonstrates redirect-following
```

Notes:
- `urlopen` follows up to 10 redirects automatically; the final `resp` reflects the landing URL (`resp.geturl()` if you want to see it).
- `HTTPError` (4xx/5xx) is caught so you still get the status/headers/body instead of a traceback.
- `resp.read(500)` reads only the first 500 bytes off the socket — no need to pull the whole body.
