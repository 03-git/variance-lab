Reflex check (reflex.1, pythonâ†’kernel): the primitive here is the stdlib `urllib.request` (sits on `socket`) â€” no third-party `requests` needed, which would be `fail.additive`.

```python
#!/usr/bin/env python3
import sys
import urllib.request

def main():
    if len(sys.argv) != 2:
        print(f"usage: {sys.argv[0]} URL", file=sys.stderr)
        sys.exit(2)

    url = sys.argv[1]
    # urllib follows redirects by default (HTTPRedirectHandler).
    req = urllib.request.Request(url, headers={"User-Agent": "curl-ish/1.0"})
    try:
        with urllib.request.urlopen(req) as resp:
            body = resp.read(500).decode("utf-8", errors="replace")
            status = resp.status
            headers = resp.headers
            final_url = resp.geturl()
    except urllib.error.HTTPError as e:
        # HTTPError is also a response-like object.
        body = e.read(500).decode("utf-8", errors="replace")
        status = e.code
        headers = e.headers
        final_url = e.url

    print(f"URL (final): {final_url}")
    print(f"Status: {status}")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Notes:
- `urllib.request.urlopen` follows 3xx redirects automatically; `resp.geturl()` shows where it landed.
- `HTTPError` (4xx/5xx) is caught so you still get status/headers/body instead of a traceback.
- `resp.read(500)` reads at most 500 bytes; for multibyte UTF-8 that can be <500 chars and may clip a character â€” `errors="replace"` keeps it from throwing. If you want exactly 500 *characters*, read more bytes then slice the decoded string.
