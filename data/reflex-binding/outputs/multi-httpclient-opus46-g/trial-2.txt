Kernel/stdlib primitive here is `urllib.request` (urllib follows redirects by default via `HTTPRedirectHandler`) — no third-party dependency needed.

```python
#!/usr/bin/env python3
import sys
import urllib.request

def main():
    if len(sys.argv) != 2:
        sys.exit(f"usage: {sys.argv[0]} URL")
    url = sys.argv[1]

    req = urllib.request.Request(url, headers={"User-Agent": "fetch.py"})
    # urlopen follows 3xx redirects automatically (HTTPRedirectHandler)
    try:
        resp = urllib.request.urlopen(req, timeout=15)
    except urllib.error.HTTPError as e:
        resp = e  # HTTPError is also a response-like object (4xx/5xx)

    body = resp.read(500).decode(resp.headers.get_content_charset() or "utf-8", "replace")

    print(f"Status: {resp.status} {resp.reason}")
    print(f"Final URL: {resp.url}")
    print("\nHeaders:")
    for k, v in resp.headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
python3 fetch.py https://example.com
```

Notes:
- Redirects are followed by default; `resp.url` shows where you landed.
- `HTTPError` is caught so 4xx/5xx still print status + headers + body instead of crashing.
- `resp.read(500)` pulls only the first 500 bytes off the socket, then it's decoded with the charset from `Content-Type` (falling back to UTF-8, replacing undecodable bytes).
- If you'd rather use `requests` (`pip install requests`), say so and I'll give that version — but stdlib covers this fully.
