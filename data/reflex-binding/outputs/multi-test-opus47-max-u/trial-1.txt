Here's a pytest test suite for `celsius_to_fahrenheit`:

```python
# test_temperature.py
import pytest


def celsius_to_fahrenheit(c):
    return c * 9/5 + 32


def test_freezing_point():
    assert celsius_to_fahrenheit(0) == 32


def test_boiling_point():
    assert celsius_to_fahrenheit(100) == 212


def test_negative_forty_is_same_in_both_scales():
    assert celsius_to_fahrenheit(-40) == -40


def test_non_numeric_input_raises():
    with pytest.raises(TypeError):
        celsius_to_fahrenheit("hot")
```

How to run it:

```bash
pip install pytest
pytest test_temperature.py -v
```

Expected output: 4 passed. The `-v` flag shows each test name; drop it for compact output.

If you'd rather not depend on pytest, here's the same suite using the stdlib `unittest`:

```python
import unittest


def celsius_to_fahrenheit(c):
    return c * 9/5 + 32


class TestCelsiusToFahrenheit(unittest.TestCase):
    def test_freezing_point(self):
        self.assertEqual(celsius_to_fahrenheit(0), 32)

    def test_boiling_point(self):
        self.assertEqual(celsius_to_fahrenheit(100), 212)

    def test_negative_forty_is_same_in_both_scales(self):
        self.assertEqual(celsius_to_fahrenheit(-40), -40)

    def test_non_numeric_input_raises(self):
        with self.assertRaises(TypeError):
            celsius_to_fahrenheit("hot")


if __name__ == "__main__":
    unittest.main()
```

Run with `python test_temperature.py`.
