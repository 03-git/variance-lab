I'll show you an async TCP echo server using Python's `asyncio` module. This handles multiple concurrent clients efficiently.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read up to 1024 bytes (up to a newline)
            data = await reader.readuntil(b'\n')
            
            if not data:
                break
            
            # Echo it back
            writer.write(data)
            await writer.drain()
            
            print(f"Echo to {addr}: {data.decode().strip()}")
    except asyncio.IncompleteReadError:
        # Client disconnected without sending newline
        pass
    except Exception as e:
        print(f"Error with {addr}: {e}")
    finally:
        print(f"Client disconnected: {addr}")
        writer.close()
        await writer.wait_closed()

async def main():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server started on {addr[0]}:{addr[1]}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run it:**

```bash
# Terminal 1: Start the server
python3 echo_server.py
```

```bash
# Terminal 2: Connect a client
nc localhost 8888
# or telnet localhost 8888
```

Then type text and press Enter—it echoes back immediately.

**To test with multiple clients simultaneously:**

```bash
# Terminal 2
(echo "hello"; sleep 1; echo "world") | nc localhost 8888

# Terminal 3 (at same time)
(echo "foo"; sleep 1; echo "bar") | nc localhost 8888
```

The server handles both clients concurrently without blocking. Each client gets its own `handle_client()` coroutine running concurrently via `asyncio.start_server()`.
