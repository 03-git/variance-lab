I'll show you a modern async TCP echo server using Python's `asyncio` module.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client disconnected
            if not data:
                break
            
            # Echo the data back
            writer.write(data)
            await writer.drain()
            
            print(f"Echo from {addr}: {data.decode().strip()}")
    except asyncio.CancelledError:
        pass
    finally:
        writer.close()
        await writer.wait_closed()
        print(f"Client disconnected: {addr}")

async def main():
    """Start the TCP echo server."""
    server = await asyncio.start_server(
        handle_client, '127.0.0.1', 8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Server listening on {addr}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run it:**

```bash
# Terminal 1: Start the server
python3 echo_server.py

# Terminal 2-N: Connect clients (in separate terminals)
telnet 127.0.0.1 8888
# Type anything and press Enter to see it echoed back
```

Or test with `nc` (netcat):
```bash
echo "hello" | nc 127.0.0.1 8888
echo "world" | nc 127.0.0.1 8888
```

**How it works:**
- `asyncio.start_server()` creates a server that accepts connections
- `handle_client()` runs concurrently for each connected client
- `reader.readline()` waits for a line without blocking other clients
- `writer.write()` + `writer.drain()` sends data back
- The server scales to thousands of concurrent clients efficiently since each client runs in a lightweight coroutine, not a thread
