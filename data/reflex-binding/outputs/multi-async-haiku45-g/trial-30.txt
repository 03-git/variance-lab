I'll show you a Python async TCP echo server using `asyncio`. This uses the kernel primitive (`asyncio` streams) rather than a wrapper library.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # Empty read means client closed connection
            if not data:
                break
            
            # Echo the line back
            writer.write(data)
            await writer.drain()
    except Exception as e:
        print(f"Error handling client: {e}")
    finally:
        writer.close()
        await writer.wait_closed()

async def main():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        host='127.0.0.1',
        port=8888
    )
    
    print("Echo server listening on 127.0.0.1:8888")
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run it:**

```bash
python3 echo_server.py
```

**To test it** (in another terminal):

```bash
# Using netcat
echo "Hello, server!" | nc 127.0.0.1 8888

# Or telnet for interactive testing
telnet 127.0.0.1 8888
# Then type lines and they'll echo back
```

**How it works:**
- `asyncio.start_server()` creates a server that accepts concurrent connections
- Each client gets its own `handle_client()` coroutine running concurrently
- `reader.readline()` waits for a line (non-blocking, yields to event loop)
- `writer.write()` + `writer.drain()` sends data back
- The server handles thousands of concurrent clients efficiently without threads

The `await writer.drain()` is crucial—it flushes the send buffer and yields control, allowing other clients to be served while this one's data is being transmitted.
