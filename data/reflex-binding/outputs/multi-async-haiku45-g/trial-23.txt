I'll show you an async TCP echo server using Python's `asyncio` library that handles multiple concurrent clients.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    client_addr = writer.get_extra_info('peername')
    print(f"Client connected: {client_addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client closed connection
            if not data:
                break
            
            # Echo the data back
            writer.write(data)
            await writer.drain()
            
            print(f"Echo from {client_addr}: {data.decode().strip()}")
    
    except asyncio.CancelledError:
        pass
    finally:
        writer.close()
        await writer.wait_closed()
        print(f"Client disconnected: {client_addr}")

async def main():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server listening on {addr}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run the server:**

```bash
python3 echo_server.py
```

**To test it, in another terminal:**

```bash
# Using nc (netcat)
echo "Hello, server!" | nc localhost 8888

# Or using telnet for interactive testing
telnet localhost 8888
```

**How it works:**

- `handle_client()` is called for each new connection and runs concurrently via asyncio
- `readline()` waits for a line of text from the client (non-blocking)
- `writer.write()` and `drain()` send the data back to the client
- Multiple clients can connect simultaneously — each gets its own `handle_client()` coroutine
- `start_server()` creates a server that accepts connections on localhost:8888
