I'll show you an async TCP echo server using Python's `asyncio` library that handles multiple concurrent clients.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    client_addr = writer.get_extra_info('peername')
    print(f"Client connected: {client_addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client disconnected
            if not data:
                print(f"Client disconnected: {client_addr}")
                break
            
            # Echo the data back
            writer.write(data)
            await writer.drain()
            
            # Optional: print what we echoed
            print(f"Echo to {client_addr}: {data.decode().strip()}")
    
    except Exception as e:
        print(f"Error handling {client_addr}: {e}")
    finally:
        writer.close()
        await writer.wait_closed()

async def start_server():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server running on {addr[0]}:{addr[1]}")
    
    async with server:
        await server.serve_forever()

if __name__ == "__main__":
    asyncio.run(start_server())
```

**How to run it:**

```bash
# Terminal 1: Start the server
python3 echo_server.py

# Terminal 2: Connect with netcat and test
nc localhost 8888
hello
hello
world
world
^C

# Or connect multiple clients simultaneously in separate terminals
```

**How it works:**

1. `handle_client()` — processes each connected client asynchronously
   - Reads lines with `reader.readline()`
   - Echoes them back with `writer.write()` and `await writer.drain()`
   - Handles disconnection when `readline()` returns empty bytes

2. `start_server()` — creates an async server listening on `127.0.0.1:8888`
   - Calls `handle_client` for each new connection
   - `serve_forever()` keeps it running

3. Multiple clients are handled concurrently because each `handle_client()` runs as a separate coroutine — while one client waits for input, others can be served.

To test with a Python client instead of netcat:

```python
import asyncio

async def test_client():
    reader, writer = await asyncio.open_connection('127.0.0.1', 8888)
    
    writer.write(b"Hello from async client\n")
    await writer.drain()
    
    response = await reader.readline()
    print(f"Received: {response.decode()}")
    
    writer.close()
    await writer.wait_closed()

# Run: asyncio.run(test_client())
```
