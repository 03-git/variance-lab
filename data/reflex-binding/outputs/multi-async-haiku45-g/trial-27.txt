I'll show you an async TCP echo server using Python's `asyncio` module that handles multiple concurrent clients efficiently.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line of text from the client
            data = await reader.readline()
            
            # If client disconnects, data will be empty
            if not data:
                print(f"Client disconnected: {addr}")
                break
            
            # Echo the data back to the client
            writer.write(data)
            await writer.drain()
            
            # Log the echo
            print(f"Echo to {addr}: {data.decode().strip()}")
    
    except Exception as e:
        print(f"Error with {addr}: {e}")
    finally:
        writer.close()
        await writer.wait_closed()

async def main():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server started on {addr}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run it:**

```bash
# Terminal 1: Start the server
python3 echo_server.py
```

**To test it (Terminal 2+):**

```bash
# Connect with netcat
nc localhost 8888

# Then type messages and press Enter — they'll echo back:
# > Hello world
# Hello world
# > Test message
# Test message
```

Or use `telnet`:
```bash
telnet localhost 8888
```

**How it works:**

- `asyncio.start_server()` creates a server listening on `127.0.0.1:8888`
- `handle_client()` is called for each new connection
- `reader.readline()` waits for a line of input (non-blocking, allows other clients to be served)
- `writer.write()` sends the data back, and `await writer.drain()` ensures it's flushed
- When a client closes the connection, `readline()` returns empty bytes and we exit the loop
- Multiple clients can connect simultaneously and each runs its handler concurrently

The key advantage of `asyncio` is that a single thread can handle thousands of concurrent clients without threads or processes—each client's handler yields control while waiting for I/O, letting others progress.
