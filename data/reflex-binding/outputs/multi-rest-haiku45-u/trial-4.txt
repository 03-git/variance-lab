I'll show you a REST API endpoint in Python using Flask. This is a simple, production-ready implementation with proper validation.

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json()
    
    # Validate required fields
    if not data:
        return jsonify({'error': 'Request body must be JSON'}), 400
    
    errors = {}
    
    # Validate name
    if 'name' not in data:
        errors['name'] = 'name is required'
    elif not isinstance(data['name'], str) or not data['name'].strip():
        errors['name'] = 'name must be a non-empty string'
    
    # Validate email
    if 'email' not in data:
        errors['email'] = 'email is required'
    elif not isinstance(data['email'], str) or '@' not in data['email']:
        errors['email'] = 'email must be a string containing @'
    
    # Validate age
    if 'age' not in data:
        errors['age'] = 'age is required'
    elif not isinstance(data['age'], int) or data['age'] < 1 or data['age'] > 150:
        errors['age'] = 'age must be an integer between 1 and 150'
    
    if errors:
        return jsonify({'errors': errors}), 400
    
    # Return validated user on success
    user = {
        'id': 1,  # In production, generate or use a database
        'name': data['name'],
        'email': data['email'],
        'age': data['age']
    }
    return jsonify(user), 201

if __name__ == '__main__':
    app.run(debug=True, port=5000)
```

**To run it:**

1. Install Flask:
   ```bash
   pip install flask
   ```

2. Save the code to a file (e.g., `app.py`) and run:
   ```bash
   python app.py
   ```

3. Test with curl:

   **Valid request (201):**
   ```bash
   curl -X POST http://localhost:5000/users \
     -H "Content-Type: application/json" \
     -d '{"name":"Alice","email":"alice@example.com","age":30}'
   ```

   **Invalid request (400):**
   ```bash
   curl -X POST http://localhost:5000/users \
     -H "Content-Type: application/json" \
     -d '{"name":"Bob","email":"invalid-email","age":200}'
   ```

The endpoint validates all three fields and returns detailed error messages for each invalid field, or a 201 with the validated user object on success.
