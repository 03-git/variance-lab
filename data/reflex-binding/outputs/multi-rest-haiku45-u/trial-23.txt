I'll show you a REST API endpoint implementation in Python using Flask. Here's the complete code:

```python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/users', methods=['POST'])
def create_user():
    data = request.get_json()
    
    # Validate that JSON was provided
    if data is None:
        return jsonify({'error': 'Request body must be valid JSON'}), 400
    
    # Validate name
    name = data.get('name')
    if not name or not isinstance(name, str):
        return jsonify({'error': 'name is required and must be a string'}), 400
    if not name.strip():
        return jsonify({'error': 'name cannot be empty'}), 400
    
    # Validate email
    email = data.get('email')
    if not email or not isinstance(email, str):
        return jsonify({'error': 'email is required and must be a string'}), 400
    if '@' not in email:
        return jsonify({'error': 'email must contain @'}), 400
    
    # Validate age
    age = data.get('age')
    if age is None:
        return jsonify({'error': 'age is required'}), 400
    if not isinstance(age, int) or isinstance(age, bool):
        return jsonify({'error': 'age must be an integer'}), 400
    if age < 1 or age > 150:
        return jsonify({'error': 'age must be between 1 and 150'}), 400
    
    # Return validated user
    user = {
        'name': name.strip(),
        'email': email,
        'age': age
    }
    return jsonify(user), 201

if __name__ == '__main__':
    app.run(debug=True, port=5000)
```

**To run it:**

```bash
pip install flask
python app.py
```

The server will start on `http://localhost:5000`.

**Example requests:**

```bash
# Valid request
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "John Doe", "email": "john@example.com", "age": 30}'

# Missing required field
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Jane", "age": 25}'

# Invalid email (no @)
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Jane", "email": "invalid-email", "age": 25}'

# Age out of range
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Jane", "email": "jane@example.com", "age": 200}'
```

The endpoint returns 400 with error details on validation failure, and 201 with the validated user object on success.
