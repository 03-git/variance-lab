```python
import pytest

def celsius_to_fahrenheit(c):
    return c * 9/5 + 32

def test_freezing():
    assert celsius_to_fahrenheit(0) == 32.0

def test_boiling():
    assert celsius_to_fahrenheit(100) == 212.0

def test_negative_forty():
    # -40 is where Celsius and Fahrenheit converge
    assert celsius_to_fahrenheit(-40) == -40.0

def test_non_numeric_raises():
    with pytest.raises(TypeError):
        celsius_to_fahrenheit("hot")
```

**To run it:**

1. Save the code to a file, e.g. `test_temp.py`
2. Install pytest if needed: `pip install pytest`
3. Run: `pytest test_temp.py -v`

Expected output:
```
test_temp.py::test_freezing        PASSED
test_temp.py::test_boiling         PASSED
test_temp.py::test_negative_forty  PASSED
test_temp.py::test_non_numeric_raises PASSED
```

The `-40` test is a useful sanity check since it's the only temperature where both scales are equal. The `TypeError` test relies on Python's native behavior — multiplying a string by `9/5` raises `TypeError` without any extra validation needed.
