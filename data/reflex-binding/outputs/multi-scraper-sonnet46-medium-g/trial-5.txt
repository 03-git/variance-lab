```python
import urllib.request
from html.parser import HTMLParser

class LinkParser(HTMLParser):
    def __init__(self):
        super().__init__()
        self.links = []
        self._current_href = None
        self._current_text = []

    def handle_starttag(self, tag, attrs):
        if tag == "a":
            attrs = dict(attrs)
            self._current_href = attrs.get("href")
            self._current_text = []

    def handle_data(self, data):
        if self._current_href is not None:
            self._current_text.append(data)

    def handle_endtag(self, tag):
        if tag == "a" and self._current_href is not None:
            text = "".join(self._current_text).strip()
            self.links.append((self._current_href, text))
            self._current_href = None
            self._current_text = []

url = "https://example.com"
with urllib.request.urlopen(url) as response:
    html = response.read().decode("utf-8")

parser = LinkParser()
parser.feed(html)

for href, text in parser.links:
    print(f"{text!r:40} -> {href}")
```

**How to run** (no dependencies beyond stdlib):

```bash
python3 script.py
```

This uses only the standard library — `urllib.request` for the HTTP fetch and `html.parser.HTMLParser` for parsing. For `https://example.com` specifically, you'll likely see one link: `"More information..."` pointing to `https://www.iana.org/domains/reserved`.

If you later need to handle relative URLs, redirect chains, or more complex pages, `urllib.parse.urljoin(url, href)` resolves relative hrefs, and `http.cookiejar` handles session state — all still stdlib.
