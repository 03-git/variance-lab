Certainly! Below is a simple example of an HTTP endpoint that returns the current time as JSON using Python and the Flask framework. The endpoint runs at `/time` and returns a JSON response with the current time in ISO 8601 format.

### Python Code

```python
from flask import Flask, jsonify
from datetime import datetime

app = Flask(__name__)

@app.route('/time', methods=['GET'])
def get_current_time():
    current_time = datetime.now().isoformat()
    return jsonify({"time": current_time})

if __name__ == '__main__':
    app.run(debug=True)
```

### Explanation

- **`from flask import Flask, jsonify`**: Imports the necessary components from the Flask framework.
- **`from datetime import datetime`**: Imports the `datetime` module to get the current date and time.
- **`app = Flask(__name__)`**: Creates a new Flask application.
- **`@app.route('/time', methods=['GET'])`**: Defines a route for the `/time` endpoint.
- **`def get_current_time()`**: The function that handles the GET request to the `/time` endpoint.
- **`current_time = datetime.now().isoformat()`**: Gets the current date and time and converts it to an ISO 8601 string.
- **`return jsonify({"time": current_time})`**: Returns the current time as a JSON object.

### Running the Code

To run this Flask application, follow these steps:

1. **Install Flask**: If you don't have Flask installed, you can install it using pip:

   ```sh
   pip install Flask
   ```

2. **Save the Code**: Save the code above in a file named `app.py`.

3. **Run the Application**: Run the application using the following command:

   ```sh
   python app.py
   ```

4. **Access the Endpoint**: Open your web browser or use a tool like `curl` or Postman to access the endpoint:

   ```sh
   curl http://127.0.0.1:5000/time
   ```

   Or, in a web browser, navigate to `http://127.0.0.1:5000/time`.

### Example Response

When you access the endpoint, you should receive a JSON response like this:

```json
{
  "time": "2023-10-01T12:34:56.789012"
}
```

This response will contain the current date and time in ISO 8601 format.

- The `debug=True` parameter is useful for development, as it enables debug mode, which provides more detailed error messages and live reloading.
- In a production environment, you should disable debug mode and consider other security measures.
