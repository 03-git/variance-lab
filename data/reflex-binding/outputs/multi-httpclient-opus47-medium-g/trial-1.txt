Kernel primitive first (reflex.1): an HTTP GET is just bytes on a TCP socket — `socket()`/`connect()`/`sendall()`/`recv()`, plus an OpenSSL handshake for `https` and manual `3xx`/`Location` chasing for redirects. Hand-rolling all that is a lot, and the stdlib `urllib.request` wraps exactly those syscalls without any third-party install — so no `requests`/`httpx`. Here's the script:

```python
#!/usr/bin/env python3
import sys
import urllib.request
import urllib.error

def main():
    if len(sys.argv) != 2:
        sys.exit(f"usage: {sys.argv[0]} <url>")
    url = sys.argv[1]

    req = urllib.request.Request(url, headers={"User-Agent": "fetch.py/1"})
    try:
        # urllib follows redirects by default via HTTPRedirectHandler
        with urllib.request.urlopen(req, timeout=15) as resp:
            status = resp.status
            headers = resp.headers
            body = resp.read(500).decode(resp.headers.get_content_charset() or "utf-8", "replace")
            final_url = resp.geturl()
    except urllib.error.HTTPError as e:
        # HTTPError is also a response — report it instead of dying
        status, headers, final_url = e.code, e.headers, e.url
        body = e.read(500).decode("utf-8", "replace")
    except urllib.error.URLError as e:
        sys.exit(f"request failed: {e.reason}")

    print(f"Final URL: {final_url}")
    print(f"Status: {status}")
    print("Headers:")
    for k, v in headers.items():
        print(f"  {k}: {v}")
    print("\nBody (first 500 chars):")
    print(body)

if __name__ == "__main__":
    main()
```

Run it:

```bash
# save as fetch.py, then:
python3 fetch.py https://example.com

# follows redirects automatically, e.g.:
python3 fetch.py http://github.com      # -> https://github.com/
```

Notes:
- `urlopen` follows `301/302/303/307` automatically; `resp.geturl()` shows where it landed.
- `HTTPError` (4xx/5xx) is caught and reported as a real response rather than crashing.
- `resp.read(500)` reads only the first 500 bytes off the socket — it won't pull a huge body.
