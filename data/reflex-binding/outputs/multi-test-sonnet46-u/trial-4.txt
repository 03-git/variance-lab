```python
# celsius_to_fahrenheit.py

def celsius_to_fahrenheit(c):
    return c * 9/5 + 32


# test_celsius_to_fahrenheit.py

import unittest
from celsius_to_fahrenheit import celsius_to_fahrenheit

class TestCelsiusToFahrenheit(unittest.TestCase):

    def test_freezing_point(self):
        self.assertEqual(celsius_to_fahrenheit(0), 32)

    def test_boiling_point(self):
        self.assertEqual(celsius_to_fahrenheit(100), 212)

    def test_equal_point(self):
        # -40 is the point where Celsius and Fahrenheit are equal
        self.assertEqual(celsius_to_fahrenheit(-40), -40)

    def test_non_numeric_input(self):
        with self.assertRaises(TypeError):
            celsius_to_fahrenheit("hot")

if __name__ == "__main__":
    unittest.main()
```

**How to run:**

1. Save both snippets into their respective files (`celsius_to_fahrenheit.py` and `test_celsius_to_fahrenheit.py`) in the same directory.

2. Run with unittest:
   ```bash
   python -m unittest test_celsius_to_fahrenheit.py -v
   ```

3. Or, since the test file has `__main__`, you can run it directly:
   ```bash
   python test_celsius_to_fahrenheit.py
   ```

**Expected output:**
```
test_boiling_point ... ok
test_equal_point ... ok
test_freezing_point ... ok
test_non_numeric_input ... ok

----------------------------------------------------------------------
Ran 4 tests in 0.001s

OK
```

A note on the `TypeError` test: Python will raise `TypeError` naturally when you try to multiply a string by a float, so no extra input validation is needed in the function to make that test pass.
