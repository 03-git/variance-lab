I'll show you a Python CLI tool that does exactly that:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(
        description="Parse host and port configuration"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=int,
        default=8000,
        help="Port number (default: 8000)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )

    try:
        args = parser.parse_args()
    except SystemExit as e:
        if e.code != 0:
            sys.exit(1)
        sys.exit(0)

    # Validate port range
    if not (1 <= args.port <= 65535):
        print(
            json.dumps({
                "error": f"Port must be between 1 and 65535, got {args.port}"
            }),
            file=sys.stderr
        )
        sys.exit(1)

    # Build config
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }

    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage
python cli.py

# With custom host and port
python cli.py --host example.com --port 3000

# With verbose flag
python cli.py --host 0.0.0.0 --port 9000 --verbose

# With invalid port (shows error)
python cli.py --port 70000
```

**Output examples:**

```json
{
  "host": "localhost",
  "port": 8000,
  "verbose": false
}
```

```json
{
  "host": "example.com",
  "port": 3000,
  "verbose": true
}
```

The argparse module automatically handles type conversion and the port validation catches out-of-range values with a clear error message.
