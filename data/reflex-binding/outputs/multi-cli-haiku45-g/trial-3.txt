I'll show you a Python CLI tool with argument validation and JSON output:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(
        description="Print parsed configuration as JSON"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=int,
        default=8080,
        help="Port number (default: 8080)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    # Validate port range
    if not (1 <= args.port <= 65535):
        print(
            f"Error: port must be between 1 and 65535, got {args.port}",
            file=sys.stderr
        )
        sys.exit(1)
    
    # Build config dict
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    # Output as JSON
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage (uses defaults)
python script.py

# With custom host and port
python script.py --host example.com --port 3000

# With verbose flag
python script.py --host 192.168.1.1 --port 9000 --verbose

# Invalid port (will error)
python script.py --port 99999
```

**Example outputs:**

```bash
$ python script.py --host api.local --port 5000 --verbose
{
  "host": "api.local",
  "port": 5000,
  "verbose": true
}

$ python script.py --port 0
Error: port must be between 1 and 65535, got 0
```

The script uses `argparse` for clean flag handling, validates the port range after parsing, and exits with status 1 if validation fails.
