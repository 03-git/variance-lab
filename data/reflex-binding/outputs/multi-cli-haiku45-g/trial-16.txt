Here's a Python CLI tool that does what you need:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(description="Configuration parser CLI")
    parser.add_argument("--host", default="localhost", help="Host address (default: localhost)")
    parser.add_argument("--port", type=int, default=8000, help="Port number (default: 8000)")
    parser.add_argument("--verbose", action="store_true", help="Enable verbose output")
    
    args = parser.parse_args()
    
    # Validate port
    if not (1 <= args.port <= 65535):
        print(f"Error: Port must be between 1 and 65535, got {args.port}", file=sys.stderr)
        sys.exit(1)
    
    # Build and output config as JSON
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save as config_cli.py, then:

# Basic usage (defaults)
python config_cli.py

# With custom host and port
python config_cli.py --host 192.168.1.1 --port 3000

# With verbose flag
python config_cli.py --host example.com --port 9000 --verbose

# Invalid port (exits with error)
python config_cli.py --port 99999

# All flags
python config_cli.py --host 0.0.0.0 --port 5000 --verbose
```

**Example outputs:**

```json
# Default
{
  "host": "localhost",
  "port": 8000,
  "verbose": false
}

# With custom values and verbose
{
  "host": "example.com",
  "port": 3000,
  "verbose": true
}
```

The `argparse` module handles type conversion and the `--verbose` flag is a boolean. The port validation rejects anything outside 1-65535 and exits with code 1 on error.
