Here's a Python CLI tool that meets your requirements:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(description='Parse and validate host/port configuration')
    parser.add_argument('--host', default='localhost', help='Host to connect to (default: localhost)')
    parser.add_argument('--port', type=int, help='Port number (1-65535)')
    parser.add_argument('--verbose', action='store_true', help='Enable verbose output')
    
    args = parser.parse_args()
    
    # Validate port if provided
    if args.port is not None:
        if args.port < 1 or args.port > 65535:
            print(json.dumps({
                'error': f'Port must be between 1 and 65535, got {args.port}'
            }), file=sys.stderr)
            sys.exit(1)
    
    config = {
        'host': args.host,
        'port': args.port,
        'verbose': args.verbose
    }
    
    print(json.dumps(config, indent=2))

if __name__ == '__main__':
    main()
```

**Usage examples:**

```bash
# Basic usage with defaults
python cli.py
# Output:
# {
#   "host": "localhost",
#   "port": null,
#   "verbose": false
# }

# With all flags
python cli.py --host example.com --port 8080 --verbose
# Output:
# {
#   "host": "example.com",
#   "port": 8080,
#   "verbose": true
# }

# Invalid port (too high)
python cli.py --port 70000
# Error output:
# {"error": "Port must be between 1 and 65535, got 70000"}
# Exit code: 1

# Invalid port (not a number)
python cli.py --port abc
# Error: argument --port: invalid int value: 'abc'
```

The tool uses `argparse` for clean argument parsing, validates the port range, and outputs valid JSON to stdout (errors go to stderr). Non-zero exit code on validation failure.
