I'll write a Python CLI tool with argument validation and JSON output.

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(
        description="CLI tool for host and port configuration"
    )
    parser.add_argument(
        "--host",
        required=True,
        help="Host address"
    )
    parser.add_argument(
        "--port",
        type=int,
        required=True,
        help="Port number (1-65535)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    # Validate port range
    if not (1 <= args.port <= 65535):
        print(
            json.dumps({
                "error": f"Port must be between 1 and 65535, got {args.port}"
            }),
            file=sys.stderr
        )
        sys.exit(1)
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage
python tool.py --host localhost --port 8080

# Output: {"host": "localhost", "port": 8080, "verbose": false}

# With verbose flag
python tool.py --host 0.0.0.0 --port 3000 --verbose

# Output: {"host": "0.0.0.0", "port": 3000, "verbose": true}

# Invalid port (too high)
python tool.py --host localhost --port 99999

# Output to stderr: {"error": "Port must be between 1 and 65535, got 99999"}
# Exit code: 1

# Invalid port (not an integer)
python tool.py --host localhost --port abc

# argparse error: argument --port: invalid int value: 'abc'
```

The tool uses `argparse` for clean argument parsing, validates the port range with a check after parsing, and outputs valid configs as JSON to stdout. Invalid ports print an error JSON object to stderr and exit with code 1.
