I'll write a Python CLI tool with argument parsing and validation:

```python
import argparse
import json
import sys


def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
        if not (1 <= port <= 65535):
            raise argparse.ArgumentTypeError(
                f"port must be between 1 and 65535, got {port}"
            )
        return port
    except ValueError:
        raise argparse.ArgumentTypeError(f"port must be an integer, got {value}")


def main():
    parser = argparse.ArgumentParser(
        description="Parse and display host/port configuration"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8080,
        help="Port number (1-65535, default: 8080)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )

    args = parser.parse_args()

    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }

    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage with defaults
python cli.py
# Output: {"host": "localhost", "port": 8080, "verbose": false}

# With custom host and port
python cli.py --host example.com --port 3000
# Output: {"host": "example.com", "port": 3000, "verbose": false}

# With verbose flag
python cli.py --host api.example.com --port 443 --verbose
# Output: {"host": "api.example.com", "port": 443, "verbose": true}

# Invalid port (shows error)
python cli.py --port 70000
# ArgumentTypeError: port must be between 1 and 65535, got 70000

# Non-integer port (shows error)
python cli.py --port abc
# ArgumentTypeError: port must be an integer, got abc
```

The `validate_port()` function is a custom type converter for argparse that handles validation. It returns the validated integer or raises an `ArgumentTypeError` with a descriptive message if validation fails.
