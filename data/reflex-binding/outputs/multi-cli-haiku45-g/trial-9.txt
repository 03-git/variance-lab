Here's a Python CLI tool that meets your requirements:

```python
#!/usr/bin/env python3
import argparse
import json
import sys

def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
    except ValueError:
        raise argparse.ArgumentTypeError(f"port must be an integer, got '{value}'")
    
    if not (1 <= port <= 65535):
        raise argparse.ArgumentTypeError(f"port must be between 1-65535, got {port}")
    
    return port

def main():
    parser = argparse.ArgumentParser(
        description="Parse and display host/port configuration as JSON"
    )
    parser.add_argument("--host", type=str, default="localhost", help="Host to connect to (default: localhost)")
    parser.add_argument("--port", type=validate_port, default=8080, help="Port number 1-65535 (default: 8080)")
    parser.add_argument("--verbose", action="store_true", help="Enable verbose output")
    
    args = parser.parse_args()
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage (uses defaults)
python3 cli_tool.py
# Output: {"host": "localhost", "port": 8080, "verbose": false}

# With custom host and port
python3 cli_tool.py --host example.com --port 3000
# Output: {"host": "example.com", "port": 3000, "verbose": false}

# With verbose flag
python3 cli_tool.py --host 0.0.0.0 --port 5432 --verbose
# Output: {"host": "0.0.0.0", "port": 5432, "verbose": true}

# Invalid port (too high)
python3 cli_tool.py --port 99999
# Error: usage: cli_tool.py [-h] [--host HOST] [--port PORT] [--verbose]
#        cli_tool.py: error: argument --port: port must be between 1-65535, got 99999

# Invalid port (not an integer)
python3 cli_tool.py --port abc
# Error: usage: cli_tool.py [-h] [--host HOST] [--port PORT] [--verbose]
#        cli_tool.py: error: argument --port: port must be an integer, got 'abc'
```

The `validate_port()` function uses argparse's custom type validation to check both that the value is an integer and that it's within the valid port range (1-65535). Any validation failure raises an `ArgumentTypeError` which argparse handles automatically with a clear error message.
