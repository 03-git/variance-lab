I'll show you a Python CLI tool with argument parsing, validation, and JSON output:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(
        description="CLI tool to configure host and port settings"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=int,
        default=8080,
        help="Port number (default: 8080)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    # Validate port range
    if not (1 <= args.port <= 65535):
        print(f"Error: port must be between 1 and 65535, got {args.port}", file=sys.stderr)
        sys.exit(1)
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save as config_tool.py, then:
python config_tool.py --host example.com --port 3000 --verbose

# Output:
# {
#   "host": "example.com",
#   "port": 3000,
#   "verbose": true
# }

# With defaults:
python config_tool.py

# Output:
# {
#   "host": "localhost",
#   "port": 8080,
#   "verbose": false
# }

# Invalid port (triggers error):
python config_tool.py --port 70000
# Error: port must be between 1 and 65535, got 70000
```

The `argparse.ArgumentParser` handles type validation (port must be int), the conditional checks the 1-65535 range, and `json.dumps()` with `indent=2` formats the output.
