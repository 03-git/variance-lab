Here's a Python CLI tool with argument validation:

```python
import argparse
import json
import sys


def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
    except ValueError:
        raise argparse.ArgumentTypeError(f"'{value}' is not a valid integer")
    
    if not (1 <= port <= 65535):
        raise argparse.ArgumentTypeError(f"port must be between 1 and 65535, got {port}")
    
    return port


def main():
    parser = argparse.ArgumentParser(
        description="Parse host and port configuration"
    )
    parser.add_argument("--host", required=True, help="Host address")
    parser.add_argument("--port", type=validate_port, required=True, help="Port (1-65535)")
    parser.add_argument("--verbose", action="store_true", help="Enable verbose output")
    
    args = parser.parse_args()
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage
python script.py --host localhost --port 8080

# With verbose flag
python script.py --host 0.0.0.0 --port 3000 --verbose

# Invalid port (fails validation)
python script.py --host localhost --port 99999
# Error: port must be between 1 and 65535, got 99999

# Missing required argument
python script.py --host localhost
# Error: the following arguments are required: --port
```

**Example output:**
```json
{
  "host": "localhost",
  "port": 8080,
  "verbose": false
}
```

The validation happens at parse time via the `validate_port` function, which raises an `ArgumentTypeError` if the port is invalid. This prevents any invalid configuration from being processed.
