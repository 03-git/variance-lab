Here's a Python CLI tool with argument parsing, port validation, and JSON output:

```python
import argparse
import json


def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
        if not 1 <= port <= 65535:
            raise ValueError(f"port must be between 1 and 65535, got {port}")
        return port
    except ValueError as e:
        raise argparse.ArgumentTypeError(str(e))


def main():
    parser = argparse.ArgumentParser(
        description="Parse and display configuration as JSON"
    )
    parser.add_argument(
        "--host",
        type=str,
        default="localhost",
        help="Host to connect to (default: localhost)",
    )
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8000,
        help="Port number (1-65535, default: 8000)",
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output",
    )

    args = parser.parse_args()

    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose,
    }

    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Default values
python cli.py
# Output:
# {
#   "host": "localhost",
#   "port": 8000,
#   "verbose": false
# }

# With custom host and port
python cli.py --host 0.0.0.0 --port 3000
# Output:
# {
#   "host": "0.0.0.0",
#   "port": 3000,
#   "verbose": false
# }

# With verbose flag
python cli.py --host api.example.com --port 443 --verbose
# Output:
# {
#   "host": "api.example.com",
#   "port": 443,
#   "verbose": true
# }

# Invalid port (validation fails)
python cli.py --port 70000
# Error: argument --port: port must be between 1 and 65535, got 70000
```

The `validate_port()` function is a custom type converter for argparse that ensures the port is an integer in the valid range. If validation fails, argparse automatically displays a helpful error message.
