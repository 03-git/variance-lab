Kernel primitive: `unittest` from the stdlib — no third-party test runner needed.

```python
# test_temp.py
import unittest


def celsius_to_fahrenheit(c):
    return c * 9 / 5 + 32


class TestCelsiusToFahrenheit(unittest.TestCase):
    def test_freezing(self):
        self.assertEqual(celsius_to_fahrenheit(0), 32)

    def test_boiling(self):
        self.assertEqual(celsius_to_fahrenheit(100), 212)

    def test_minus_forty(self):
        # the one temperature where C and F coincide
        self.assertEqual(celsius_to_fahrenheit(-40), -40)

    def test_non_numeric(self):
        with self.assertRaises(TypeError):
            celsius_to_fahrenheit("hot")


if __name__ == "__main__":
    unittest.main()
```

Run it:

```
python -m unittest test_temp.py -v
```

Or, since it has the `__main__` guard:

```
python test_temp.py -v
```

Notes:
- `"hot" * 9` actually succeeds in Python (string repetition), but the subsequent `/ 5` raises `TypeError` — so the non-numeric test passes. If you'd rather fail fast, add an explicit `isinstance` check to the function and the test still holds.
- Float cases like `celsius_to_fahrenheit(37)` → `98.6` would want `assertAlmostEqual`; the four values you asked for are all exact.
