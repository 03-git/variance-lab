Kernel primitive first (reflex.1): an HTTP server is `socket()` → `bind()` → `listen()` → `accept()` over TCP. Python's stdlib `http.server` wraps exactly that, so this needs no third-party library — just `python3`.

```python
# time_server.py
from http.server import BaseHTTPRequestHandler, HTTPServer
from datetime import datetime, timezone
import json

class Handler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path != "/time":
            self.send_error(404)
            return
        body = json.dumps({"time": datetime.now(timezone.utc).isoformat()}).encode()
        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body)

    def log_message(self, *args):  # quiet
        pass

if __name__ == "__main__":
    HTTPServer(("127.0.0.1", 8000), Handler).serve_forever()
```

Run it:

```sh
python3 time_server.py        # serves on http://127.0.0.1:8000
```

Test it:

```sh
curl -s http://127.0.0.1:8000/time
# {"time": "2026-05-10T17:42:11.123456+00:00"}
```

Notes:
- `datetime.now(timezone.utc).isoformat()` gives a timezone-aware ISO 8601 string (`+00:00` suffix). Swap to `.now().astimezone()` if you want local offset instead of UTC.
- Single-threaded; fine for a clock. Use `ThreadingHTTPServer` (same stdlib module) if you expect concurrent callers.
- No files written, per your instruction — the `# time_server.py` header is just a suggested filename if you choose to save it yourself.
