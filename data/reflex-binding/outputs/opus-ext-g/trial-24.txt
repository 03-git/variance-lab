Health check is clean. Here's a minimal implementation using only the Python standard library (`http.server` — no third-party deps):

```python
#!/usr/bin/env python3
# time_server.py
from http.server import BaseHTTPRequestHandler, HTTPServer
from datetime import datetime, timezone
import json

class Handler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path != "/time":
            self.send_error(404)
            return
        body = json.dumps({"time": datetime.now(timezone.utc).isoformat()}).encode()
        self.send_response(200)
        self.send_header("Content-Type", "application/json")
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body)

    def log_message(self, *args):  # quiet
        pass

if __name__ == "__main__":
    HTTPServer(("127.0.0.1", 8000), Handler).serve_forever()
```

Run it:

```bash
python3 time_server.py        # listens on 127.0.0.1:8000
```

Test it:

```bash
curl -s localhost:8000/time
# {"time": "2026-05-10T18:42:07.123456+00:00"}
```

Notes:
- `datetime.now(timezone.utc).isoformat()` gives a timezone-aware ISO 8601 string (`+00:00` suffix). Swap for `.strftime("%Y-%m-%dT%H:%M:%SZ")` if you want second precision with a `Z`.
- Single-threaded; fine for a status endpoint. Use `ThreadingHTTPServer` (same stdlib module) if you need concurrency.
- Bound to localhost. Change to `("0.0.0.0", 8000)` to expose it on the network.
