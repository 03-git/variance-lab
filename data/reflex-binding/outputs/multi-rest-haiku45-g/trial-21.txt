I'll show you a complete REST API implementation in Python using Flask. This handles all your requirements with proper validation.

```python
from flask import Flask, request, jsonify
from datetime import datetime

app = Flask(__name__)

def validate_user(data):
    """Validate user input and return (is_valid, user_dict_or_errors)"""
    errors = {}
    
    # Validate name
    if 'name' not in data:
        errors['name'] = 'name is required'
    elif not isinstance(data['name'], str) or not data['name'].strip():
        errors['name'] = 'name must be a non-empty string'
    
    # Validate email
    if 'email' not in data:
        errors['email'] = 'email is required'
    elif not isinstance(data['email'], str) or '@' not in data['email']:
        errors['email'] = 'email must be a string containing @'
    
    # Validate age
    if 'age' not in data:
        errors['age'] = 'age is required'
    elif not isinstance(data['age'], int) or data['age'] < 1 or data['age'] > 150:
        errors['age'] = 'age must be an integer between 1 and 150'
    
    if errors:
        return False, errors
    
    return True, {
        'id': datetime.now().timestamp(),
        'name': data['name'].strip(),
        'email': data['email'].strip(),
        'age': data['age']
    }

@app.route('/users', methods=['POST'])
def create_user():
    try:
        data = request.get_json()
        if data is None:
            return jsonify({'error': 'Request body must be valid JSON'}), 400
    except Exception as e:
        return jsonify({'error': 'Invalid JSON format'}), 400
    
    is_valid, result = validate_user(data)
    
    if not is_valid:
        return jsonify({'errors': result}), 400
    
    return jsonify(result), 201

if __name__ == '__main__':
    app.run(debug=True, port=5000)
```

**To run it:**

```bash
# Install Flask if needed
pip install flask

# Save the code to api.py and run it
python api.py
```

The server starts on `http://localhost:5000`.

**Test it with curl:**

```bash
# Valid request
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Alice", "email": "alice@example.com", "age": 30}'

# Missing name (400 response)
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"email": "bob@example.com", "age": 25}'

# Invalid email (missing @)
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Bob", "email": "invalid-email", "age": 25}'

# Invalid age (outside range)
curl -X POST http://localhost:5000/users \
  -H "Content-Type: application/json" \
  -d '{"name": "Charlie", "email": "charlie@example.com", "age": 200}'
```

The endpoint returns a 201 status with the validated user object on success, or 400 with detailed error messages for any validation failures.
