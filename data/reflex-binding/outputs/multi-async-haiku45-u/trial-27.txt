I'll show you an async TCP echo server using Python's `asyncio` module.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client closed the connection
            if not data:
                break
            
            # Echo the data back to the client
            writer.write(data)
            await writer.drain()
            
            # Print what we received
            message = data.decode().strip()
            print(f"[{addr}] Echoed: {message}")
    
    except Exception as e:
        print(f"Error handling client {addr}: {e}")
    finally:
        writer.close()
        await writer.wait_closed()
        print(f"Client disconnected: {addr}")


async def start_server(host='127.0.0.1', port=9999):
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        host,
        port
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server listening on {addr}")
    
    async with server:
        await server.serve_forever()


if __name__ == '__main__':
    asyncio.run(start_server())
```

**How to run it:**

1. **Start the server** (in one terminal):
```bash
python echo_server.py
```

2. **Connect a client** (in another terminal):
```bash
nc localhost 9999
```
or using telnet:
```bash
telnet localhost 9999
```

3. **Send messages**: Type any text and press Enter. The server will echo it back. You can open multiple terminals and connect multiple clients simultaneously—they'll all be handled concurrently.

**Example interaction:**
```
$ nc localhost 9999
hello world
hello world
testing async
testing async
^C
```

The server uses `asyncio.start_server()` which automatically creates a new task for each client via `handle_client()`. This lets it handle unlimited concurrent clients efficiently without threading.
