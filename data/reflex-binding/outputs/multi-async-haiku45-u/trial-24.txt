I'll show you a clean async TCP echo server using Python's `asyncio` module.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            if not data:
                break
            
            message = data.decode().strip()
            print(f"[{addr}] Received: {message}")
            
            # Echo it back
            writer.write(data)
            await writer.drain()
            
    except asyncio.CancelledError:
        pass
    finally:
        print(f"Client disconnected: {addr}")
        writer.close()
        await writer.wait_closed()

async def main():
    """Start the echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server listening on {addr}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**How to run it:**

1. Save the code to a file (e.g., `echo_server.py`)
2. Run the server:
   ```bash
   python echo_server.py
   ```
3. In separate terminal(s), connect with `nc` or `telnet`:
   ```bash
   nc localhost 8888
   ```
   Then type lines and they'll be echoed back.

**Key features:**

- **`asyncio.start_server()`** — creates a TCP server that handles multiple clients concurrently
- **`handle_client()`** — async coroutine that reads lines and echoes them back for each connected client
- **`reader.readline()`** — reads until `\n`, non-blocking
- **`writer.drain()`** — flushes the write buffer without blocking
- **Concurrency** — each client runs its own `handle_client()` coroutine, so they don't block each other

The server handles 10+ simultaneous clients without issue—asyncio multiplexes them all on a single thread.
