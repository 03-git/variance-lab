I'll show you an async TCP echo server using Python's `asyncio` module. This will handle multiple concurrent clients efficiently.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client disconnected
            if not data:
                break
            
            # Echo the data back
            writer.write(data)
            await writer.drain()
            print(f"Echo to {addr}: {data.decode().strip()}")
    
    except asyncio.CancelledError:
        pass
    finally:
        print(f"Client disconnected: {addr}")
        writer.close()
        await writer.wait_closed()

async def start_server(host='127.0.0.1', port=9999):
    """Start the echo server."""
    server = await asyncio.start_server(handle_client, host, port)
    addr = server.sockets[0].getsockname()
    print(f"Echo server running on {addr[0]}:{addr[1]}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(start_server())
```

**To run the server:**

```bash
python echo_server.py
```

**To test it with multiple clients (in separate terminals):**

```bash
# Terminal 1: Start the server
python echo_server.py

# Terminal 2: Connect with netcat and send data
nc localhost 9999
# Type anything and press Enter, you'll see it echoed back
hello
# Output: hello

# Terminal 3: Another client (runs concurrently)
nc localhost 9999
world
# Output: world
```

**How it works:**

- `asyncio.start_server()` creates a server that accepts connections and calls `handle_client()` for each one
- `reader.readline()` reads a line from the client asynchronously
- `writer.write()` + `writer.drain()` sends data back to the client
- Multiple clients are handled concurrently without threads—the event loop multiplexes I/O operations
- The server runs indefinitely until interrupted with Ctrl+C

You can also use `telnet localhost 9999` or write a Python client script to connect instead of netcat.
