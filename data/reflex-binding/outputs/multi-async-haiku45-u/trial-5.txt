I'll show you an async TCP echo server using Python's `asyncio` module that handles multiple concurrent clients efficiently.

```python
import asyncio

async def handle_client(reader, writer):
    """Handle a single client connection."""
    addr = writer.get_extra_info('peername')
    print(f"Client connected: {addr}")
    
    try:
        while True:
            # Read a line from the client
            data = await reader.readline()
            
            # If no data, client disconnected
            if not data:
                break
            
            # Echo the data back
            writer.write(data)
            await writer.drain()
            
            print(f"Echo from {addr}: {data.decode().strip()}")
    
    except Exception as e:
        print(f"Error handling {addr}: {e}")
    finally:
        writer.close()
        await writer.wait_closed()
        print(f"Client disconnected: {addr}")

async def main():
    """Start the TCP echo server."""
    server = await asyncio.start_server(
        handle_client,
        '127.0.0.1',
        8888
    )
    
    addr = server.sockets[0].getsockname()
    print(f"Echo server listening on {addr}")
    
    async with server:
        await server.serve_forever()

if __name__ == '__main__':
    asyncio.run(main())
```

**To run it:**

```bash
# Terminal 1: Start the server
python echo_server.py

# Terminal 2, 3, etc: Connect clients and send lines
nc localhost 8888
# or use telnet
telnet localhost 8888
```

**How it works:**
- `asyncio.start_server()` creates a TCP server that accepts connections
- For each client, `handle_client()` runs concurrently (no blocking)
- `reader.readline()` reads one line at a time, waiting asynchronously
- `writer.write()` + `writer.drain()` sends data back to the client
- Each client connection is handled independently without threads

**To test with multiple clients:**
```bash
# In separate terminals, connect and type:
echo "Hello" | nc localhost 8888
echo "World" | nc localhost 8888
```

The server handles unlimited concurrent clients efficiently because `asyncio` multiplexes them on a single thread—no threads or process overhead needed.
