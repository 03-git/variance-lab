Here's a Python script using the `requests` library:

```python
import sys
import requests

def fetch_url(url):
    try:
        response = requests.get(url, allow_redirects=True, timeout=10)
        
        print(f"Status Code: {response.status_code}")
        print("\nResponse Headers:")
        for header, value in response.headers.items():
            print(f"  {header}: {value}")
        
        print(f"\nBody (first 500 characters):")
        print(response.text[:500])
        
    except requests.exceptions.RequestException as e:
        print(f"Error: {e}", file=sys.stderr)
        sys.exit(1)

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python script.py <url>", file=sys.stderr)
        sys.exit(1)
    
    fetch_url(sys.argv[1])
```

**To run it:**

```bash
python script.py https://example.com
```

**Key features:**
- `allow_redirects=True` follows redirects automatically
- `timeout=10` prevents hanging on slow servers
- Prints status code, all headers, and first 500 chars of response body
- Error handling for network issues

**If you don't have `requests` installed:**
```bash
pip install requests
```

Alternatively, here's a stdlib version using `urllib` (no dependencies):

```python
import sys
import urllib.request
import urllib.error

def fetch_url(url):
    try:
        response = urllib.request.urlopen(url, timeout=10)
        
        print(f"Status Code: {response.status}")
        print("\nResponse Headers:")
        for header, value in response.headers.items():
            print(f"  {header}: {value}")
        
        body = response.read().decode('utf-8', errors='replace')
        print(f"\nBody (first 500 characters):")
        print(body[:500])
        
    except urllib.error.URLError as e:
        print(f"Error: {e}", file=sys.stderr)
        sys.exit(1)

if __name__ == "__main__":
    if len(sys.argv) < 2:
        print("Usage: python script.py <url>", file=sys.stderr)
        sys.exit(1)
    
    fetch_url(sys.argv[1])
```

The `requests` version is simpler and more robust—I'd recommend that one.
