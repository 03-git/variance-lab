Kernel primitive: an HTTP GET over a TCP socket + an HTML tokenizer. Python's stdlib covers both (`urllib.request`, `html.parser`) — no third-party libs needed.

```python
#!/usr/bin/env python3
"""Fetch a URL and print every <a href> with its link text."""
import sys
from urllib.request import urlopen, Request
from html.parser import HTMLParser


class LinkExtractor(HTMLParser):
    def __init__(self):
        super().__init__()
        self.links = []          # list of [href, text]
        self._href = None        # href of the <a> currently open

    def handle_starttag(self, tag, attrs):
        if tag == "a":
            self._href = dict(attrs).get("href")
            if self._href is not None:
                self.links.append([self._href, ""])

    def handle_data(self, data):
        if self._href is not None and self.links:
            self.links[-1][1] += data.strip()

    def handle_endtag(self, tag):
        if tag == "a":
            self._href = None


def main(url):
    req = Request(url, headers={"User-Agent": "link-extractor/1.0"})
    with urlopen(req, timeout=10) as resp:
        charset = resp.headers.get_content_charset() or "utf-8"
        html = resp.read().decode(charset, errors="replace")

    parser = LinkExtractor()
    parser.feed(html)

    for href, text in parser.links:
        print(f"{href}\t{text or '(no text)'}")


if __name__ == "__main__":
    main(sys.argv[1] if len(sys.argv) > 1 else "https://example.com")
```

Run it:

```bash
python3 script.py                 # defaults to https://example.com
python3 script.py https://...     # or pass any URL
```

On `https://example.com` you'll get a single line — that page has exactly one link:

```
https://www.iana.org/domains/example	More information...
```

Notes:
- Pure stdlib; nothing to `pip install`.
- `html.parser` is lenient but not a full DOM — fine for extracting tags, not for malformed-HTML edge cases. If you ever need that robustness, `beautifulsoup4` + `lxml` is the usual wrapper, but it's not warranted here.
- No redirect/HTTPS-verification fuss: `urlopen` follows redirects and verifies TLS via the system trust store by default.
