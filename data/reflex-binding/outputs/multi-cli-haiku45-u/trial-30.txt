I'll show you a Python CLI tool that handles those requirements:

```python
import argparse
import json
import sys


def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
        if not (1 <= port <= 65535):
            raise ValueError(f"port must be between 1 and 65535, got {port}")
        return port
    except ValueError as e:
        raise argparse.ArgumentTypeError(str(e))


def main():
    parser = argparse.ArgumentParser(
        description="CLI tool that prints parsed configuration as JSON"
    )
    parser.add_argument("--host", default="localhost", help="Host to connect to")
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8080,
        help="Port number (1-65535)",
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output",
    )

    args = parser.parse_args()

    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose,
    }

    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage
python cli.py

# With custom host and port
python cli.py --host 192.168.1.1 --port 3000

# With verbose flag
python cli.py --host example.com --port 5000 --verbose

# Invalid port (will error)
python cli.py --port 70000
```

**Example outputs:**

```bash
$ python cli.py
{
  "host": "localhost",
  "port": 8080,
  "verbose": false
}

$ python cli.py --host 0.0.0.0 --port 9000 --verbose
{
  "host": "0.0.0.0",
  "port": 9000,
  "verbose": true
}
```

The tool uses `argparse` for argument parsing, a custom `validate_port()` function as a type converter to enforce the 1-65535 range, and outputs the final configuration as JSON.
