I'll show you a Python CLI tool that does exactly that:

```python
import argparse
import json
import sys

def main():
    parser = argparse.ArgumentParser(
        description="CLI tool to validate and display host/port configuration"
    )
    
    parser.add_argument(
        "--host",
        type=str,
        default="localhost",
        help="Host address (default: localhost)"
    )
    
    parser.add_argument(
        "--port",
        type=int,
        default=8080,
        help="Port number (default: 8080)"
    )
    
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    # Validate port range
    if not (1 <= args.port <= 65535):
        print(f"Error: port must be between 1 and 65535, got {args.port}", file=sys.stderr)
        sys.exit(1)
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save to a file first (e.g., cli_tool.py), then:

# Basic usage with defaults
python cli_tool.py

# With custom host and port
python cli_tool.py --host example.com --port 3000

# With verbose flag
python cli_tool.py --host 0.0.0.0 --port 9000 --verbose

# Invalid port (demonstrates validation)
python cli_tool.py --port 99999
# Error: port must be between 1 and 65535, got 99999
```

The tool uses `argparse` to handle the CLI arguments, validates the port is within the valid range (1-65535), and outputs the configuration as formatted JSON.
