Here's a Python CLI tool that does what you need:

```python
#!/usr/bin/env python3
import argparse
import json
import sys


def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
    except ValueError:
        raise argparse.ArgumentTypeError(f"Port must be an integer, got '{value}'")
    
    if not (1 <= port <= 65535):
        raise argparse.ArgumentTypeError(
            f"Port must be between 1 and 65535, got {port}"
        )
    return port


def main():
    parser = argparse.ArgumentParser(
        description="Parse and display configuration as JSON"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8000,
        help="Port number (1-65535, default: 8000)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save as cli_tool.py and make executable:
chmod +x cli_tool.py

# Basic usage (defaults to localhost:8000):
python3 cli_tool.py

# With custom host and port:
python3 cli_tool.py --host example.com --port 3000

# With verbose flag:
python3 cli_tool.py --host 0.0.0.0 --port 5432 --verbose

# Invalid port (shows error):
python3 cli_tool.py --port 99999
# Error: usage: cli_tool.py [-h] [--host HOST] [--port PORT] [--verbose]
# cli_tool.py: error: argument --port: Port must be between 1 and 65535, got 99999

# Help:
python3 cli_tool.py --help
```

**Example outputs:**

```json
// Default
{"host": "localhost", "port": 8000, "verbose": false}

// With args
{"host": "example.com", "port": 3000, "verbose": true}
```

The script uses `argparse.ArgumentTypeError` to provide clear validation messages, and `action="store_true"` for the boolean flag.
