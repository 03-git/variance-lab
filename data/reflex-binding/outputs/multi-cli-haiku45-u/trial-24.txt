I'll create a Python CLI tool with argument parsing and validation:

```python
import argparse
import json
import sys

def validate_port(value):
    """Validate that port is an integer between 1-65535."""
    try:
        port = int(value)
        if not (1 <= port <= 65535):
            raise argparse.ArgumentTypeError(
                f"port must be between 1 and 65535, got {port}"
            )
        return port
    except ValueError:
        raise argparse.ArgumentTypeError(f"port must be an integer, got '{value}'")

def main():
    parser = argparse.ArgumentParser(
        description="CLI tool to parse and display host/port configuration"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host to connect to (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8080,
        help="Port number (default: 8080)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))
    
    if args.verbose:
        print(f"[DEBUG] Configuration loaded successfully", file=sys.stderr)

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save as cli_tool.py, then:

# Basic usage (defaults)
python cli_tool.py

# With custom host and port
python cli_tool.py --host example.com --port 3000

# With verbose flag
python cli_tool.py --host 0.0.0.0 --port 9000 --verbose

# Invalid port (will show error)
python cli_tool.py --port 99999
python cli_tool.py --port abc
```

**Example outputs:**

```bash
$ python cli_tool.py --host api.example.com --port 5432 --verbose
{
  "host": "api.example.com",
  "port": 5432,
  "verbose": true
}
[DEBUG] Configuration loaded successfully

$ python cli_tool.py --port 0
usage: cli_tool.py [-h] [--host HOST] [--port PORT] [--verbose]

```

The tool uses `argparse` for clean argument handling, a custom `validate_port()` function that enforces the 1-65535 range, and outputs the parsed config as formatted JSON.
