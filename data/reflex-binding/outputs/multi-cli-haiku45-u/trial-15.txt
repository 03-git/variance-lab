Here's a Python CLI tool that meets your requirements:

```python
import argparse
import json

def validate_port(value):
    """Validate that port is an integer between 1-65535"""
    try:
        port = int(value)
        if not (1 <= port <= 65535):
            raise ValueError(f"port must be between 1 and 65535, got {port}")
        return port
    except ValueError as e:
        raise argparse.ArgumentTypeError(str(e))

def main():
    parser = argparse.ArgumentParser(
        description="CLI tool that validates and displays host/port configuration"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host address (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=validate_port,
        default=8000,
        help="Port number (1-65535, default: 8000)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )
    
    args = parser.parse_args()
    
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }
    
    print(json.dumps(config, indent=2))

if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Save as config_cli.py and run with different options:

# Default values
python config_cli.py
# Output:
# {
#   "host": "localhost",
#   "port": 8000,
#   "verbose": false
# }

# Custom host and port
python config_cli.py --host 0.0.0.0 --port 3000
# Output:
# {
#   "host": "0.0.0.0",
#   "port": 3000,
#   "verbose": false
# }

# With verbose flag
python config_cli.py --host example.com --port 443 --verbose
# Output:
# {
#   "host": "example.com",
#   "port": 443,
#   "verbose": true
# }

# Invalid port (shows error)
python config_cli.py --port 70000
# Error: argument --port: port must be between 1 and 65535, got 70000

# Non-integer port (shows error)
python config_cli.py --port abc
# Error: argument --port: port must be between 1 and 65535, got abc
```

The `validate_port()` function acts as a custom type converter for argparse—it runs when the `--port` argument is parsed and raises an `ArgumentTypeError` if validation fails. This gives you clean error messages out of the box.
