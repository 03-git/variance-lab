I'll create a Python CLI tool with argparse that validates the flags and outputs JSON.

```python
import argparse
import json
import sys


def main():
    parser = argparse.ArgumentParser(
        description="CLI tool to parse and validate host/port configuration"
    )
    parser.add_argument(
        "--host",
        default="localhost",
        help="Host address (default: localhost)"
    )
    parser.add_argument(
        "--port",
        type=int,
        default=8000,
        help="Port number (default: 8000)"
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output"
    )

    try:
        args = parser.parse_args()
    except SystemExit:
        sys.exit(1)

    # Validate port range
    if not (1 <= args.port <= 65535):
        print(
            json.dumps({
                "error": f"Port must be between 1 and 65535, got {args.port}"
            }),
            file=sys.stderr
        )
        sys.exit(1)

    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose
    }

    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Basic usage (uses defaults)
python script.py

# With custom host and port
python script.py --host example.com --port 3000

# With verbose flag
python script.py --host 0.0.0.0 --port 9000 --verbose

# Invalid port (error to stderr, exit code 1)
python script.py --port 70000

# Non-integer port (argparse handles this)
python script.py --port abc
```

**Example outputs:**

```json
{
  "host": "localhost",
  "port": 8000,
  "verbose": false
}
```

```json
{
  "host": "example.com",
  "port": 3000,
  "verbose": true
}
```

The tool uses `argparse` for flag parsing, validates that port is an integer and in the valid range (1-65535), and outputs the configuration as formatted JSON.
