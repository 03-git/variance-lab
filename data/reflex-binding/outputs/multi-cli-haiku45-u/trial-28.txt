I'll show you a Python CLI tool that uses `argparse` to handle the flags and validation:

```python
import argparse
import json
import sys


def main():
    parser = argparse.ArgumentParser(
        description="Print parsed configuration as JSON"
    )
    parser.add_argument("--host", required=True, help="Hostname or IP address")
    parser.add_argument(
        "--port",
        type=int,
        required=True,
        help="Port number (1-65535)",
    )
    parser.add_argument(
        "--verbose",
        action="store_true",
        help="Enable verbose output",
    )

    try:
        args = parser.parse_args()
    except SystemExit:
        sys.exit(1)

    # Validate port range
    if not (1 <= args.port <= 65535):
        print(
            json.dumps({"error": f"Port must be between 1 and 65535, got {args.port}"}),
            file=sys.stderr,
        )
        sys.exit(1)

    # Build configuration
    config = {
        "host": args.host,
        "port": args.port,
        "verbose": args.verbose,
    }

    print(json.dumps(config, indent=2))


if __name__ == "__main__":
    main()
```

**How to run it:**

```bash
# Valid example
python script.py --host localhost --port 8080

# With verbose flag
python script.py --host 192.168.1.1 --port 3000 --verbose

# Invalid port (too high)
python script.py --host localhost --port 70000

# Invalid port (not an integer)
python script.py --host localhost --port abc
```

**Example outputs:**

```json
{"host": "localhost", "port": 8080, "verbose": false}
```

```json
{"host": "192.168.1.1", "port": 3000, "verbose": true}
```

The tool validates the port as an integer automatically (argparse raises an error if it can't parse), then checks it's in the valid range (1-65535) and outputs the config as JSON.
