`install.sh` is a no-privilege installer that sets up the subtract system under `~/.subtract/`. Here's what it does:

**Detects execution mode** (lines 1-16): runs in `local` mode if executed from within the repo (detects `runtime/` sibling dir), otherwise `web` mode (piped via `curl`).

**Verifies the manifest signature before installing anything** (lines 30-40): in web mode, fetches `llms.txt`, its `.sig`, and `authorized_signers`, then runs `ssh-keygen -Y verify` against `hodori@subtract.ing` in the `subtract.ing` namespace. Aborts with a warning if verification fails.

**Creates the directory structure**: `~/.subtract/`, `hooks/`, `bin/`, `log/clinical/`.

**Fetches runtime files**: copies or curls the subtract shell (`subtract.sh`), hooks (`bash.sh`, `zsh.sh`), lookdown tables, and browser shell tools (`shell-web`, `subtract-shell`, `browse`).

**Integrates with the shell** (lines 95-104): appends a source line for the appropriate hook (`bash.sh` or `zsh.sh`) and `PATH` export to `~/.bashrc` and/or `~/.zshrc`, if not already present.

The core guarantee: nothing runs until `ssh-keygen -Y verify` passes on the manifest. That's the authority gate from `governance.conf.universal` applied at install time.
