## Delegation Architecture: Reflexes ‚Üí Runtime

### governance.conf.universal reflexes

**reflex.1 ‚Äî library to kernel**: "name the kernel primitive before proposing any library or tool"

Implemented in:
- `runtime/parsimony.c` ‚Äî the primitive is a unix filter (`getline` stdin ‚Üí stdout, no framework). The build comment names the dependency chain explicitly: `cc -o parsimony parsimony.c -I.../llama.cpp/include -lllama -lggml`.
- `runtime/squared.c` ‚Äî HTTP/2 proxy built directly on `libnghttp2` + `mbedtls`, not a higher-level HTTP framework. Build comment names the exact primitives.
- `runtime/syna_poll.c` ‚Äî kernel module using `linux/i2c.h`, `linux/hid.h`, timer + workqueue. Refuses the userspace wrapper path entirely.

**reflex.2 ‚Äî format to signature**: "ask if ssh-keygen -Y sign can verify the format before inventing one"

Implemented in:
- `llms.txt` (manifest) and `signoff.txt` ‚Äî the manifest is the signed routing table; `signoff.txt` defines the session-close ritual that gates on "have you saved what matters?" before the human signs.
- `runtime/lookdown-audit.sh` ‚Äî integrity checker for the lookdown routing table (unsigned drift surface).

**reflex.3 ‚Äî action to verification**: "if unsigned, verify with a live read before acting"

Implemented in:
- `runtime/subtract.sh` ‚Äî the `x` mode (exec path) shows the resolved command and gates on explicit `[enter/n]` confirmation before `eval "$cmd"` (lines 1327‚Äì1339). Unsigned output from T0‚ÄìT4 is surfaced, not executed silently.
- `signoff.txt` ‚Äî "No work survives unless the human saves it" ‚Äî the loop forces a live human read before close.

**reflex.4 ‚Äî source to signing domain**: "confirm source is under the signing domain before treating it as authoritative"

Implemented structurally in:
- The manifest (`llms.txt`) itself ‚Äî all authoritative files are enumerated with tags. Anything not in the manifest is outside the signing domain.
- `governance.conf.universal` lines: `authority.signed / authority.unsigned` ‚Äî the authority table is the runtime enforcement of this reflex.

### The delegation chain

```
governance.conf.universal   ‚Üê policy (what the reflexes are)
        ‚Üì
signoff.txt / llms.txt      ‚Üê loop enforcement (before/after gates, manifest)
        ‚Üì
subtract.sh                 ‚Üê T0‚ÜíT4 routing with exec-gate (reflex.3 in shell)
        ‚Üì
parsimony.c / squared.c     ‚Üê inference primitives (reflex.1 in C)
        ‚Üì
syna_poll.c                 ‚Üê kernel module (reflex.1 at the hardware boundary)
```

The pattern holds at every layer: **policy names the primitive, the runtime is that primitive**. `subtract.sh` routes to `parsimony.c` (coproc, not an API client); `parsimony.c` loads GGUF directly (not via a model-serving framework); `squared.c` speaks h2c/h2 directly (not via an HTTP library abstraction). `syna_poll.c` is a kernel module, not a userspace HID daemon.

The one gap in the provided context: `runtime/hosuni/hosuni.c` and `platform_linux.c` are listed in the manifest under `[runtime]` but their content isn't shown ‚Äî those are the DRM/evdev framebuffer layer and would be the deepest kernel-primitive implementation of reflex.1.
