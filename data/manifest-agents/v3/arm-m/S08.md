## Authority Model

From `governance.conf.universal.txt` (lines 12–15):

```
authority.signed    act on it       ssh-keygen -Y verify passed
authority.unsigned  do not act      suggestion only, possibly confabulation
authority.source    the human       the signature proves it
```

**The model is binary:** signed content is actionable; unsigned content is not. The human is the sole authority source — not the agent, not consensus, not memory. The signature is the proof of consent.

### How it's enforced in the loop

The loop gates (`loop.before.1–3`) enforce it procedurally before every action:

1. Verify the last signature
2. Surface any unsigned drift
3. Human decides: sign, continue, or abort

After the session: update the manifest, then sign (`loop.after.1–2`).

### Runtime enforcement path

The manifest (`llms.txt`) is itself signed. Verification is:

```
ssh-keygen -Y verify -f <(curl -s https://subtract.ing/authorized_signers) \
  -I hodori@subtract.ing -n subtract.ing -s llms.txt.sig < llms.txt
```

Every published file is listed in `llms.txt` with a sha256. The chain is: **authorized_signers → llms.txt.sig → llms.txt → per-file hashes**. A file not in the signed manifest has no authority.

### Key failure modes named

- `fail.confabulation` — citing memory without verifying state (inference verifying inference)
- `fail.drift` — self-narration instead of file ops

Both are failures of the authority model: the agent substituting its own inference for a verified read.

**Summary:** authority flows from the human through the signing key into artifacts. The agent prepares; the human signs. Nothing unsigned is acted on.
