`squared.c` is an **HTTP/2 proxy for local inference multiplexing** (`squared.c` lines 1–3).

It proxies between clients and a backend inference server (e.g., llama-server). It does not perform numerical computation — the name refers to HTTP/**2** squared (h2).

**What it does:**
- Accepts connections and multiplexes them over a single TCP connection using HTTP/2 (`nghttp2`)
- Forwards each stream as a separate HTTP/1.1 connection to a backend host:port
- Supports two modes: **h2c** (cleartext, loopback) and **h2 over TLS** (ALPN-negotiated, via `mbedtls`)

**Why it exists** (from `variance-lab.txt` lines 841–856):
> "The browser is the exception. Browsers enforce a 6-connection-per-origin limit. h2 multiplexes all requests on one connection, bypassing the limit."

At concurrency ≤ 6, h2 and h1.1 show equivalent wall time because backend inference dominates. The proxy earns its keep specifically for browser clients hitting the 6-connection limit.

**Scale:** 716 lines of C, one translation unit, three libraries: `nghttp2`, `mbedtls`, `libc`.
