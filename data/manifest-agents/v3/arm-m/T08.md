The lookdown system is defined across these files:

**Schema/format** — `./lookdown.example.tsv` (lines 1-28) documents the format:
> `glob_pattern<TAB>command` or `glob_pattern<TAB>[tag]<TAB>command`

Tags are `[stdout]` (default) or `[player]`. First match wins.

**Tables:**
- `./lookdown.universal.tsv` — 2,624-line universal intent table, loaded on every node. Children's entries go here first ("every node should answer a child's question. the village helps").
- `./runtime/lookdown.m2mini.tsv` — node-specific overrides for Emile (M2 Mini). Checked *before* universal.
- `~/.subtract/lookdown.<hostname>.tsv` — per-host local overrides (not published, templated by `lookdown.example.tsv`).

**Load order** (from `lookdown-audit.sh`, lines 11-18):
1. Node-specific file (`lookdown.<hostname>.tsv`)
2. Universal file (`lookdown.universal.tsv`)

First match wins across both files combined.

**Runtime integration** — `hosuni.c` loads both tables at startup (`hosuni.txt` lines 41-48):
> "hosuni.c loads both the per-host (`~/.subtract/lookdown.<hostname>.tsv`) and universal (`~/.subtract/lookdown.universal.tsv`) tables at startup."

In READ mode, matching patterns answer instantly without inference. In RUN mode, `[player]`-tagged entries appear as launchable programs (F3 type-to-filter).

**Integrity checker** — `./runtime/bin/lookdown-audit.sh` detects shadows: entries in universal that are unreachable because a node-specific pattern matches first.
