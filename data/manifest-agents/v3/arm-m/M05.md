## Components using llama.cpp / local inference

### 1. `parsimony.c` — core inference filter
The primary local inference component. Loads a GGUF model once, reads JSON requests from stdin, writes responses to stdout. Wraps llama.cpp directly:

```c
llama_backend_init();
ggml_backend_load_all();
struct llama_model *model = llama_model_load_from_file(model_path, mp);
```

Wire format is OpenAI chat-completions-shaped. Managed as a shell `coproc`; stateless between requests (KV cache clears). Accepts `--prelude-file` for a standing system prompt (e.g., `SOUL.txt`).

### 2. `hosuni.c` / `hosuni.h` / `platform_linux.c` — framebuffer frontend
The DRM/evdev UI layer. Per the manifest it is the "framebuffer computing" runtime. It connects to local inference via parsimony — `hosuni.c` sends queries; parsimony replies over the coproc pipe. `SOUL.txt` is the prelude injected at parsimony startup for the kids' interface.

### 3. `hosuni.sh` — inference launcher
Shell wrapper (`runtime/hosuni.sh`) that launches the inference stack. Starts parsimony with the appropriate model and wires it to hosuni.c.

### 4. `squared.c` — HTTP/2 multiplexing proxy
Sits in front of `llama-server` (not parsimony) for browser-facing inference. The variance-lab finding (`variance-lab.txt`, HTTP/2 finding) showed h2 multiplexing eliminates the browser 6-connection-per-origin limit, giving 2.5× speedup at c=3 from a browser. From curl, no advantage — inference time dominates.

### 5. `nba` wrapper (Rousseau runtime, not in manifest but documented)
Routes to four local llama-server instances on Rousseau:
- `:8081` — phi-4-mini (`fast`)
- `:8082` — qwen3-32b (`rag`)
- `:8083` — gemma-4-26b (`reasoning`)
- `:8084` — Qwen3-Coder-30B-A3B (`code`)

### Relationship

```
hosuni.c (framebuffer UI)
    └── coproc → parsimony.c (llama.cpp GGUF loader, unix filter)
                     └── SOUL.txt prelude at startup

browser / hosuni-discord.ts
    └── HTTP → squared.c (h2 proxy)
                  └── HTTP → llama-server (llama.cpp HTTP API)
                                └── nba wrapper selects model/port

hosuni-discord.sh → hosuni-discord.ts → squared or direct llama-server
```

**parsimony** is the direct llama.cpp embed (no HTTP, coproc pipe). **squared** is the HTTP/2 front for llama-server. They serve different access paths: framebuffer terminal uses parsimony; browser and Discord use the HTTP stack through squared.
