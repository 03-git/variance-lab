## Lookdown → hosuni.c: call path

### 1. Startup: table loading

From `hosuni.txt` lines 41-50:
> "hosuni.c loads both the per-host (`~/.subtract/lookdown.<hostname>.tsv`) and universal (`~/.subtract/lookdown.universal.tsv`) tables at startup."

Both files are loaded into memory before any user input is processed.

### 2. User input: RUN mode (F3)

From `hosuni.txt` lines 28-31:
> "RUN (F3) — Type-to-filter launcher over the lookdown table. Every `[player]` entry in lookdown.tsv becomes a launchable program."

The user types characters; hosuni.c filters the loaded lookdown entries in real time against the typed string.

### 3. Pattern matching

From `lookdown.universal.tsv` lines 1-6, the format is:
```
glob_pattern<TAB>[tag]<TAB>command
```
Tags are `[stdout]` (default) or `[player]`. The comment says **"first match wins. specific before general."** — glob patterns are evaluated in file order against the user's input.

### 4. READ mode interception

From `hosuni.txt` lines 43-47:
> "In READ mode, lookdown **intercepts queries that match a pattern and answers instantly without inference**."

So in F1/READ mode, before a query goes to the inference endpoint, hosuni.c checks the lookdown tables. A match short-circuits the network call entirely — the matched `echo "..."` command runs locally and its stdout is displayed as the answer.

### 5. Command execution

The `[stdout]` tag means the command's stdout is printed as the response. The `[player]` tag means the command launches a program (a media player, game, etc.) as a child process. From `hosuni.c:1-40`, the file includes `<sys/wait.h>` and `<sys/socket.h>` — child process management is built in at the platform level.

### 6. Fallthrough to inference

If no lookdown pattern matches in READ mode, hosuni.c forwards the query to the inference endpoint read from `~/.subtract/inference_host` / `~/.subtract/inference_port` (default `127.0.0.1:8087`), optionally after running `hosuni-prefetch` to enrich the prompt.

---

**Summary path:**

```
keystroke → platform_event() → hosuni.c input handler
  → RUN mode: type-to-filter over loaded lookdown table → exec [player] command
  → READ mode: glob-match lookdown first
      match  → run echo command, stream stdout to display
      no match → hosuni-prefetch (optional) → inference endpoint → stream tokens
```
