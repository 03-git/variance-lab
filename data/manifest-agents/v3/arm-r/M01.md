Several components share the glob-match-then-dispatch pattern:

**1. `runtime/subtract.sh` — `__subtract_lookup_file()`**
The shell implementation of the same pattern. It lowercases input, reads a TSV, and glob-matches using `[[ "$input_lower" == $~pattern_lower ]]` (zsh) or `$pattern_lower` (bash). First match wins, tag determines handler. Identical architecture to hosuni's C implementation.

**2. `runtime/hosuni/hosuni.c` — `lookdown_check()`**
The canonical C implementation. Lowercases input, iterates `lookdown_table[]`, calls `fnmatch()` for glob matching, then `popen()` to execute the matched command. Returns output length; caller decides what to do with it.

**3. `runtime/hosuni.sh` — `check_lookdown()`**
The shell daemon version. Uses `case "$input" in $pattern)` for glob matching — bash/zsh native glob dispatch. Checks node-specific file first, then universal, returning the first matching command.

**4. `runtime/hosuni/hosuni.c` — `run_entry_visible()` / `run_entry_index()`**
The "run" mode launcher applies a secondary filter (`run_match_filter`) over the same `lookdown_table[]` — same table, different dispatch path (exec instead of popen-to-output).

**Structural invariants across all four:**
- Input lowercased before matching
- TSV source: `pattern<TAB>command`
- First match wins
- Node-specific file checked before universal

From `variance-lab.txt`: the subtract.sh T0.5 apropos handler (`"show "[0-9]*`, `"man "[0-9]*`) also fits — it matches a pattern, reads a cached result file, and dispatches to `man`. Same shape, different lookup store.
