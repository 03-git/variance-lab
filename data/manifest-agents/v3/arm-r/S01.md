## Delegation Architecture: governance.conf.universal → Runtime

### The Four Reflexes (from `governance.conf.universal.txt`)

```
reflex.1  library to kernel     name the kernel primitive before proposing any library
reflex.2  format to signature   ask if ssh-keygen -Y sign works before inventing a format
reflex.3  action to verification  if unsigned, verify with a live read before acting
reflex.4  source to signing domain  confirm source is under the signing domain
```

---

### Reflex.1 — Library to Kernel

**Spec:** `governance.conf.universal.txt` row `reflex.1`

**Runtime enforcement:** `lookdown.universal.tsv` — every row dispatches to a man-page-documented primitive (`tar`, `curl`, `rsync`, `awk`, etc.) with no framework wrapping. The routing table *is* the reflex: no library is reachable without first naming the primitive it wraps.

**Agent instruction surface:** `llms.txt` — *"Before proposing any library, any SaaS call, any framework: check if the filesystem and built-in commands already do it."*

---

### Reflex.2 — Format to Signature

**Spec:** `governance.conf.universal.txt` row `reflex.2`

**Runtime enforcement:** `install.sh` — manifest verification gate:
```sh
ssh-keygen -Y verify -f "$SCRIPT_DIR/authorized_signers" \
    -I hodori@subtract.ing -n subtract.ing \
    -s "$SCRIPT_DIR/llms.txt.sig" < "$SCRIPT_DIR/llms.txt"
```
The install script aborts on verification failure: *"ABORT: manifest signature verification failed."* This operationalizes the reflex — no new content format is trusted without asking whether `ssh-keygen -Y sign` can verify it.

---

### Reflex.3 — Action to Verification

**Spec:** `governance.conf.universal.txt` row `reflex.3`

**Runtime enforcement:** `install.sh` (live fetch + verify before any file is written). Also enforced at the loop layer:
```
loop.before.1  verify last signature   before the agent acts
loop.before.2  surface unsigned drift  before the agent acts
```

**Documented failure case:** `variance-lab.txt` — a Claude session acted on `/tmp/cal.c` (stale, 1117 lines) instead of the canonical path (1833 lines), destroying 20+ hours of work. This is explicitly named as `fail.confabulation` — "citing memory without verifying state."

The CLAUDE.md governance patch added: *"run `git log --oneline -5` … State the timestamps and line counts you observe. Only then proceed."*

---

### Reflex.4 — Source to Signing Domain

**Spec:** `governance.conf.universal.txt` row `reflex.4`

**Runtime enforcement:** `install.sh` — the `authorized_signers` file establishes the signing domain (`hodori@subtract.ing`). Any file not verifiable against that key is outside the domain.

**State surface:** `llms.txt` SHA256 manifest — canonical hashes for every served file. The manifest itself is signed; the manifest *is* the signing domain boundary.

**`subtract.7.txt`** makes this structural: *"subtract is versionless. Identity is the signing key at https://subtract.ing/authorized_signers. State is the signed manifest at https://subtract.ing/llms.txt. Verification is the only release channel."*

---

### Authority Model Summary

| Layer | File | Role |
|---|---|---|
| Specification | `governance.conf.universal.txt` | Gates as rows, not prose |
| Routing table | `lookdown.universal.tsv` | Reflex.1 operationalized |
| Deploy gate | `install.sh` | Reflex.2/3/4 enforced at install time |
| State index | `llms.txt` | Signed manifest = domain boundary |
| Agent instruction | `llms.txt` (preamble) | Reflexes re-stated for agent readers |
| Human framing | `subtract.7.txt`, `subtracting.html` | Same four reflexes in prose |

The boundary rule from `governance.conf.universal.txt` — *"the agent prepares, the human signs"* — is enforced by the fact that `install.sh` will abort rather than proceed on a failed `ssh-keygen -Y verify`. No agent path bypasses the signature gate.
