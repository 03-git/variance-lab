## parsimony.c — inference as a unix filter

`parsimony.c` is a **long-running inference daemon** that loads a GGUF model once and operates as a stateless line-oriented filter between the shell and llama.cpp.

### Wire format

**Input** (stdin, one JSON line per request):
```json
{"messages":[{"role":"...","content":"..."},...],"max_tokens":N}
```
OpenAI chat-completions-shaped. Also accepts `"stream":true`.

**Output** (stdout):
- Non-streaming: one JSON line per response
- Streaming: SSE-style `data:` lines, terminated with `data: [DONE]`

### How it fits in the runtime

`hosuni.sh` spawns parsimony as a **coproc** via two named pipes (`$SUBTRACT_DIR/.parsimony.in` and `.parsimony.out`). When both pipes are present, `hosuni.sh` writes directly to them rather than going through `curl` to a network endpoint. A mutex directory (`.parsimony.lock`) serializes concurrent callers.

`parsimony.c` also handles:
- **Prelude/SOUL injection**: loads `--prelude-file` (e.g. `SOUL.txt`) and prepends it as a system message
- **Tool injection**: if tools are present, appends the tool schema and prompt to the system message
- **Gemma tool-call parsing**: detects `{"name":..., "arguments":...}` shaped responses and re-emits them as OpenAI-style `tool_calls` objects
- **Stateless KV**: clears between requests; conversation state is owned by the orchestrator (`hosuni.sh`)

### The design principle

The comment at the top of the file states it plainly: *"The shell manages the lifecycle via coproc. Stateless: KV cache clears between requests; the orchestrator owns conversation state."* parsimony is the primitive — a thin C wrapper around llama.cpp that speaks JSON over pipes. hosuni.sh is the orchestrator.
