Based on the provided chunks, several components share the manifest/index-reading dispatch pattern:

**1. `runtime/subtract.sh` — skills index (`__subtract_skills_stale`, `__subtract_skills`)**
Reads a flat-file index at `$SUBTRACT_SKILLS/.index` (token → filename mapping), intersects candidates across search tokens, then dispatches to the matched `.md` file. The index is rebuilt when any `.md` is newer than it.

**2. `runtime/subtract.sh` — T0 lookup table (`__subtract_lookup_file`)**
Reads `lookdown.tsv` line-by-line, glob-matches the lowercased input against `pattern` column, extracts a `[tag]` and `cmd`, then dispatches by tag.

**3. `runtime/hosuni/hosuni.c` — lookdown table (RUN mode)**
Loads `lookdown_table[]` at startup, then `run_entry_visible()` filters by `LOOKDOWN_EXEC` flag and filter string. `run_entry_index()` finds the nth matching entry and dispatches it. Same glob-match-then-execute pattern as the shell layer, implemented in C.

**4. `runtime/hosuni.sh` — `check_lookdown()`**
Reads `$LOOKDOWN_NODE` and `$LOOKDOWN_UNIVERSAL` tab-delimited files, glob-matches input against `pattern`, returns the `cmd` for dispatch.

**5. `variance-lab/data/manifest-agents/rag-index.sh`**
Builds an FTS5 SQLite index from the repo's source files (chunked, 40 lines + 5 overlap). The index is then read by a RAG agent to decide what content to retrieve — same read-index-then-dispatch pattern, but at query time rather than build time.

The common structure across all five: **build or load an index → match input against it → dispatch to the resolved target**. The lookdown TSV pattern (`subtract.sh` T0, `hosuni.sh`, `hosuni.c`) is the most direct family; the skills index adds a two-level indirection (token index → file path → content); the RAG index adds FTS5 scoring on top.
