## Lookdown call path in hosuni.c

### 1. Table load (startup)

`lookdown_reload()` is called on startup and on file-watch events:

```c
// hosuni.c — lookdown_reload()
snprintf(ld_path, ..., "%s/.subtract/lookdown.%s.tsv", home, hn);  // node-specific
lookdown_load(ld_path);
snprintf(ld_path, ..., "%s/.subtract/lookdown.universal.tsv", home); // universal
lookdown_load(ld_path);
```

`lookdown_load()` parses each TSV line into `lookdown_table[]`. Tab-delimited: `pattern\tcommand`. If the command starts with `[player]` or `[exec]`, the entry gets `flags = LOOKDOWN_EXEC`; otherwise flags=0. Source file and line number are stored for provenance.

### 2. Match check (on user input)

`lookdown_check()` is the lookup function:

```c
// hosuni.c — lookdown_check()
for (i = 0; input[i] ...; i++)
    lower[i] = tolower(input[i]);   // case-fold input

for (int j = 0; j < lookdown_count; j++) {
    if (lookdown_table[j].flags & LOOKDOWN_EXEC) continue;  // skip exec entries
    if (fnmatch(lookdown_table[j].pattern, lower, 0) == 0) {
        FILE *fp = popen(lookdown_table[j].command, "r");   // run the command
        fread(output, ...);
        return len;
    }
}
```

Key detail: **`LOOKDOWN_EXEC` entries are skipped here** — `lookdown_check` only handles echo/pipe commands. The `fnmatch()` call is what makes glob patterns like `*system*call*fork*` work.

### 3. Exec path (pinned entries)

For `LOOKDOWN_EXEC` entries (flagged `[exec]` or `[player]`), execution goes through `hosuni_exec()`:

```c
// hosuni.c — hosuni_exec()
pid_t pid = fork();
if (pid == 0) {
    execlp("sh", "sh", "-c", command, NULL);
    _exit(127);
}
if (pid > 0)
    waitpid(pid, NULL, 0);
```

Signals `SIGTERM`/`SIGINT` are blocked in the parent during the child's lifetime, then unblocked after `waitpid`.

### 4. Parallel path in hosuni.sh

The shell daemon (`hosuni.sh`) has its own lookdown tier labeled **T0**, checked before any inference call:

```bash
# hosuni.sh — handle_request()
if [ "${#input}" -lt 120 ]; then        # only for short inputs
    if lookdown_result=$(check_lookdown "$input"); then
        output=$(eval "$lookdown_result" 2>&1)
        log_session "$channel" "A" "$output"
        return   # short-circuit: no inference needed
    fi
fi
```

### Summary of the full path

```
user keypress
  → handle_ask_key() builds app->ask.input
  → on Enter: lookdown_check(input, ...)
      → fnmatch() against lookdown_table[]
      → match found (non-exec): popen(command) → output displayed
      → match found (exec/pinned): hosuni_exec() → fork/exec via sh -c
      → no match: routed to inference (parsimony pipe or curl fallback)
```

The `LOOKDOWN_EXEC` flag is the branch point: static-output entries go through `popen` inside `lookdown_check`; executable/player entries go through `hosuni_exec` with a full `fork`+`waitpid` lifecycle.
