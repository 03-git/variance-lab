## Components Using Local Inference

### 1. `runtime/subtract.sh` — shell command translator + QA interface

Three functions hit local inference:

**`__subtract_generate`** (T2, command translation)
- Reads `$SUBTRACT_DIR/inference_host` and `inference_port` (default `8081`)
- If `inference_host` is remote: `ssh $host "curl http://localhost:$port/v1/chat/completions"`
- If local: hits `http://localhost:8081/v1/chat/completions` directly
- Strips markdown fences from response

**`__subtract_ask_local`** (direct QA, `ask llama.cpp` / `ask sub` / `ask local`)
- Same host/port resolution as `__subtract_generate` (default `8081`)
- Same SSH-or-local curl pattern
- Called by `ask llama.cpp <query>` and its aliases (`ask sub`, `ask local`)

**`__subtract_read`** (session-aware QA with YouTube pre-fetch)
- Same pattern but default port `8083`
- Falls back to `__subtract_ask_cloud` if local returns empty

### 2. `runtime/hosuni.sh` — Discord inference daemon

**`call_inference`** (inferred from context)
- Uses `$INFERENCE_HOST` / `$INFERENCE_PORT` env vars
- Supports a secondary `$RESEARCH_PORT` / `$RESEARCH_HOST` for a two-model handoff: primary model handles tool calls, then research model gets the assembled context with a 20s timeout (chunks 8, 10)

---

### Relationships

```
subtract.sh                         hosuni.sh
  ask sub / ask local                 Discord message
       ↓                                    ↓
  __subtract_ask_local            call_inference (primary port)
       ↓                                    ↓
  llama-server (8081)              tool call loop
  via SSH or localhost                     ↓
                                    call_inference (research port, 20s)
  __subtract_read ──────────────→ llama-server (8083, default)
  (falls back to cloud)
```

Both shells use the same wire format: OpenAI-compatible `/v1/chat/completions` JSON, `jq -r '.choices[0].message.content'` extraction. The difference is scope: `subtract.sh` is stateless per-invocation (command translation or one-shot QA); `hosuni.sh` maintains a `messages` array across rounds for multi-turn + tool use.

The shared infrastructure node is rousseau's llama-server. `subtract.sh` may SSH to a remote `inference_host` (e.g., rousseau from another node); `hosuni.sh` uses env vars that point at the same server. Both degrade gracefully — subtract falls back to cloud (`__subtract_ask_cloud`), hosuni falls back to the primary model's answer if the research handoff times out.
