Based on the retrieved chunks, here are the `.tsv` files in the subtract.ing repository:

---

### `lookdown.universal.tsv`
Glob-pattern routing table for intent-to-command translation. Format:
```
glob_pattern<TAB>command
glob_pattern<TAB>[tag]<TAB>command
```
Three-column variant adds an optional `[tag]` field between pattern and command. Input is lowercased (ASCII) before matching via `fnmatch(3)`. Contains ~2445 entries covering almanac, cooking, homework, sysadmin, clinical, etc. Listed in `llms.txt` with SHA256 `ae379026...`.

---

### `lookdown.<hostname>.tsv`
Host-specific overrides, checked **before** universal. Same format as universal. Example file: `lookdown.example.tsv` (chunk 1) and `lookdown.hodori.tsv` (rousseau's node-specific file). Contains machine-specific services (e.g. Jellyfin), hardware commands, personal shortcuts. Children's entries go in universal, not here.

---

### `lookdown.<user>.tsv`
User-specific "confessions and taste" — e.g. `lookdown.hodori.tsv` holds the governor's livingledger entries and formation-specific names. Same format as universal.

---

### `lookdown.personal.tsv`
Personal overlay, read **first** before `$SUBTRACT_LOOKUP`. Described in `variance-lab-handler-methodology.txt` as a planned modification to `__subtract_handle`. Same format. Falls through to main lookdown if file doesn't exist.

---

### `variance-lab/data/manifest-agents/manifest-walk.sh` output (not stored, generated)
Not a static `.tsv` in the repo, but `manifest-walk.sh` (chunk 7) produces:
```
path<TAB>sha256<TAB>lines<TAB>description
```

---

**Loading order at runtime** (from `hosuni.c:lookdown_reload`, chunk 11):
1. `~/.subtract/lookdown.<hostname>.tsv`
2. `~/.subtract/lookdown.universal.tsv`

First match wins.
