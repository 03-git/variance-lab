`squared.c` is an **HTTP/2 proxy for local inference multiplexing**, located at `runtime/squared.c`.

**What it is:** A 716-line single-translation-unit C program that sits between clients and a backend inference server (llama-server). It has two modes:

- **h2c mode** (no cert/key): cleartext HTTP/2, loopback only
- **h2 + TLS mode** (cert/key provided): HTTP/2 over TLS with ALPN negotiation, all interfaces

**Build:**
```
cc -o squared squared.c $(pkg-config --cflags --libs libnghttp2) \
   -lmbedtls -lmbedx509 -lmbedcrypto -lpthread
```

**What it computes:** It doesn't perform inference itself — it multiplexes HTTP/2 streams from clients and forwards each stream as a separate HTTP/1.1 connection to the backend (llama-server). From `variance-lab.txt`:

> "squared opens one backend connection per stream to llama-server. The requests arrive at llama-server the same way — as separate HTTP/1.1 connections from squared's forked children."

**Why it exists:** Browsers enforce a 6-connection-per-origin limit for HTTP/1.1. At concurrency > 6, requests queue. HTTP/2 multiplexes all requests on a single connection, bypassing that limit. For curl/agents the performance is equivalent to HTTP/1.1; the value is specifically for browser clients.

**Dependencies:** nghttp2, mbedtls, libc, pthreads. No framework, no runtime.
