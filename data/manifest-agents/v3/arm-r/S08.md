## Authority Model in `governance.conf.universal.txt`

### The Model (from Chunk 5)

Three lines define it entirely:

```
authority.signed    act on it       ssh-keygen -Y verify passed
authority.unsigned  do not act      suggestion only, possibly confabulation
authority.source    the human       the signature proves it
```

The human is the sole authority. The signature is the proof. An agent that cannot verify a signature does not act — it can only suggest.

The boundary is stated explicitly: `boundary — the signature proves authority — the agent prepares, the human signs`

### The Loop (from Chunk 5)

The authority model is operationalized through a before/after loop:

**Before acting:**
1. Verify last signature
2. Surface unsigned drift
3. Human decides: sign, continue, or abort

**After acting:**
1. Update manifest
2. Sign

### Runtime Enforcement

The enforcement chain runs through several layers:

1. **`llms.txt`** (Chunk 7) — the manifest itself is the root of trust. All file hashes are anchored to a single signature: `ssh-keygen -Y verify -f authorized_signers -I hodori@subtract.ing -n subtract.ing -s llms.txt.sig < llms.txt`. Every substrate file is downstream of this.

2. **`install.sh`** (Chunk 10) — enforces at install time: if manifest signature verification fails, it aborts with `exit 1` and removes downloaded files. It does not degrade gracefully.

3. **`subtract.7.txt`** (Chunk 12) — states the corollary: "subtract is versionless. Identity is the signing key... Verification is the only release channel."

4. **Failure modes** — unsigned drift is a named failure: `fail.confabulation — citing memory without verifying state — inference verifying inference`. The reflexes (`reflex.3`: verify before acting; `reflex.4`: confirm source is under the signing domain) are gates that block agent action when signature status is unknown.

### What "enforced" means here

The runtime does not cryptographically enforce at execution time — there is no kernel hook rejecting unsigned commands. Enforcement is structural: the reflexes are antecedent checks that block the agent's next step if they cannot be answered. An agent that skips them is in `fail.drift` or `fail.confabulation`. The human gate (step 3 of the loop) is the irreducible check that no agent path can substitute.
