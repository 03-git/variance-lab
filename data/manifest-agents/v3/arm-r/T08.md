The lookdown system spans three files:

**`runtime/hosuni/hosuni.c`** — implementation. Defines the data structure and all functions:
- `lookdown_table[]` — array of up to `MAX_LOOKDOWN` (4096) entries, each holding `pattern`, `command`, `flags`, `source_file`, `source_line`
- `LOOKDOWN_EXEC` (1) and `LOOKDOWN_PINNED` (2) flag bits
- `lookdown_load(path)` — parses a TSV file into the table; handles `[player]`/`[exec]` flag prefixes
- `lookdown_check(input, ...)` — lowercases input, runs `fnmatch()` against each non-exec entry, `popen()`s the matching command
- `pinned_save/load()` — persists pinned patterns to `~/.subtract/.pinned`
- `starred_load/save()` — persists a single starred pattern to `~/.subtract/.starred`
- `has_any_pinned()` — scans for entries with both EXEC and PINNED flags set

**`lookdown.universal.tsv`** — the data file. Tab-separated `pattern\tcommand` rows. Organized into sections by comment headers (`# === SYSCALLS ===`, `# === FILE OPERATIONS ===`, etc.). Patterns use shell glob syntax (e.g., `*system*call*fork*`). Commands are shell snippets, typically `echo "..."` for knowledge answers or bare commands like `cp`, `mv` for operations. Exec entries use `[player]\t` or `[exec]\t` prefix before the command column.

**`runtime/hosuni/hosuni.h`** — no lookdown declarations visible in the retrieved chunks; the struct and functions are local to `hosuni.c` (not exposed via the header).

The flow: `lookdown_load()` is called at startup with the TSV path → entries populate `lookdown_table` → on each input, `lookdown_check()` fnmatches against patterns in load order, first match wins, command is `popen()`'d and output returned.
