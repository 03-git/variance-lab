Based on the retrieved chunks, here are the files in `runtime/hosuni/`:

**`hosuni.c`** — Main source file. Framebuffer computing application: handles modes (READ/RUN/TEACH), conversation history, inference over raw TCP sockets (OpenAI-compatible chat completions), font rendering, gap buffer text editing, and the full application loop.

**`hosuni.h`** — Platform contract and shared types. Defines `hosuni_surface`, `hosuni_event`, `hosuni_font`, `hosuni_mode`, color constants, the gap buffer API, inference function signatures, and session log signatures. All platform implementations must satisfy the contract declared here.

**`platform_linux.c`** — Linux platform layer. DRM/KMS for display (opens `/dev/dri/card*`, double-buffered page flip with vsync) and evdev for keyboard input (`/dev/input/event*`).

**`platform_macos.m`** — macOS platform layer. Cocoa `NSWindow` with a backing bitmap for display, `NSEvent` for keyboard input. Same rendering code as Linux, different platform layer.

**`stb_truetype.h`** — Vendored single-header library for font rendering (not subtract.ing code).
