Based on the retrieved chunks:

**SOUL.txt is the system prompt for hosuni.c's inference sessions.**

From `hosuni.txt`:
> "System prompt loaded via --soul flag. Plain text file that sets the model's disposition for the conversation."

**How hosuni.c loads it** (`hosuni.c`, Chunk 1):

At startup, hosuni.c looks for SOUL.txt automatically if no `--soul` flag is given:
```c
if (!soul_path && home) {
    snprintf(default_soul, sizeof(default_soul),
             "%s/subtract.ing/SOUL.txt", home);
    if (access(default_soul, R_OK) == 0) soul_path = default_soul;
}
```
The content is read into `app.soul[16384]` (declared in the `hosuni_app` struct, Chunk 2).

**How it flows into inference** (`hosuni.c`, Chunk 8):

`build_messages_json()` prepends the soul as a `system` role message at the front of every messages array sent to the inference endpoint:
```c
if (soul && soul[0]) {
    o += snprintf(out + o, out_max - o,
        "{\"role\":\"system\",\"content\":\"%s\"}", esc);
}
```

**The override path**: You can pass `--soul /path/to/file` at launch to use a different prompt file. The default path is `~/subtract.ing/SOUL.txt`.

**One known issue** noted in `variance-lab.txt`: Sonnet 4.6 misread the `app.soul[0] = '\0'` null initializer as a bug (false positive) — the code is correct; that line just initializes the soul buffer to empty string before the file read.
