The Lua parser is implemented primarily in the following source files:

1. **lparser.c**:
   - This file contains the main functions for parsing Lua source code. It includes:
     - `mainfunc`: The main function that sets up the environment and parses the main body of the Lua code.
     - `statement`: A recursive non-terminal function for parsing statements.
     - `expr`: A recursive non-terminal function for parsing expressions.
     - `error_expected`: A function to handle syntax errors when an expected token is not found.
     - `errorlimit`: A function to handle errors related to limits in the parser.

2. **llex.h**:
   - This header file defines the token types and the structure of the lexical state (`LexState`), which is used by the parser to keep track of the current state of the input stream.

3. **lparser.h**:
   - This header file includes definitions for the parser, such as the kinds of variables and expressions (`expdesc`), and the structure of the block list (`BlockCnt`).

4. **lmem.c**:
   - Although not directly related to the parser, this file includes functions for allocating and deallocating arrays used by the parser, such as `luaM_growaux_` for growing arrays.

These files work together to provide the core functionality of the Lua parser, handling the lexical analysis, syntactic parsing, and semantic analysis of Lua source code.
