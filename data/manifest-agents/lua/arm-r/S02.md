### Key Structures and Definitions

1. **`CallInfo` Structure**:
   - Defined in `lstate.h` (lines 161-200).
   - This structure holds information about a call, including whether it's a Lua function or a C function.
   - For Lua functions, it includes fields like `savedpc` (saved program counter), `trap` (function is tracing lines/counts), and `nextraargs` (number of extra arguments in vararg functions).
   - For C functions, it includes fields like `k` (continuation in case of yields), `old_errfunc`, and `ctx` (context info. in case of yields).

2. **`lua_CFunction` and `lua_KFunction`**:
   - Defined in `lua.h` (lines 81-120).
   - `lua_CFunction` is a type for C functions registered with Lua.
   - `lua_KFunction` is a type for continuation functions.

### Relevant Functions

1. **`lua_resume` and `lua_yield`**:
   - These functions are involved in managing the state of coroutines.
   - `lua_resume` is used to resume a coroutine.
   - `lua_yield` is used to yield execution of a coroutine.

2. **`lua_pcall`**:
   - This function is used to call a Lua function in protected mode, which can be used to manage coroutines.
