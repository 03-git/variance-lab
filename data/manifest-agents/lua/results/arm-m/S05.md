Lua implements tables as a combination of a hash part and an array part. The core operations for tables are handled in the `ltable.c` and `ltable.h` files.

### Key Components and Operations:

1. **Hash Part and Array Part**:
   - Tables in Lua can store elements in two parts: an array part and a hash part.
   - Non-negative integer keys are candidates to be stored in the array part.
   - The array part is managed to keep more than half the slots between 1 and the largest used index in use.
   - The hash part uses a mix of chained scatter table with Brent's variation to handle collisions.

2. **Resizing Operations**:
   - The `luaH_resizearray` function in `ltable.c` is used to resize the array part of the table.
   - The `luaH_resize` function is a helper function used internally by `luaH_resizearray`.
   - The `luaH_resize` function first counts the keys, then resizes the table based on the new size.

3. **Hash Part Management**:
   - The `luaH_resize` function also handles the resizing of the hash part.
   - The `luaH_resizearray` function ensures that the array part is resized appropriately.
   - If the new size is smaller, it re-inserts the elements from the vanishing slice into the new hash part.

4. **Specific Functions**:
   - `luaH_resizearray` is called when the table needs to be resized, either due to an overflow or a need to reduce the size.
   - `luaH_resize` is a helper function that performs the actual resizing logic, including rehashing and reinserting elements.
