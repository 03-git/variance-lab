To trace a Lua function call from the C API through to bytecode execution, we need to follow the relevant parts of the Lua source code, particularly focusing on `lapi.c`, `ldo.c`, and `lvm.c`.

### 1. Lua Function Call via C API

Let's start with a Lua function call made through the C API. The `lua_pcall` function is a common way to call a Lua function from C. Here is the relevant part of `lapi.c`:

```c
LUA_API int lua_pcall (lua_State *L, int nargs, int nresults, int errfunc) {
  lua_lock(L);
  api_check(L, k == NULL || !isLua(L->ci),
    "cannot use continuations inside hooks");
  api_checkpop(L, nargs + 1);
  api_check(L, L->status == LUA_OK, "cannot do calls on non-normal thread");
  checkresults(L, nargs, nresults);
  if (errfunc == 0)
    func = 0;
  else {
    StkId o = index2stack(L, errfunc);
    api_check(L, ttisfunction(s2v(o)), "error handler must be a function");
    func = savestack(L, o);
  }
  c.func = L->top.p - (nargs+1);  /* function to be called */
  if (k == NULL || !yieldable(L)) {  /* no continuation or no yieldable? */
    c.nresults = nresults;  /* do a 'conventional' protected call */
    status = luaD_pcall(L, f_call, &c, savestack(L, c.func), func);
  }
  else {  /* prepare continuation (call is already protected by 'resume') */
    CallInfo *ci = L->ci;
    ci->u.c.k = k;  /* save continuation */
    ci->u.c.ctx = ctx;  /* save context */
    /* save information for error recovery */
    ci->u2.funcidx = cast_int(savestack(L, c.func));
    ci->u.c.old_errfunc = L->errfunc;
    L->errfunc = func;
    setoah(ci, L->allowhook);  /* save value of 'allowhook' */
    ci->callstatus |= CIST_YPCALL;  /* function can do error recovery */
    luaD_call(L, c.func, nresults);  /* do the call */
    ci->callstatus &= ~CIST_YPCALL;
    L->errfunc = ci->u.c.old_errfunc;
    status = LUA_OK;  /* if it is here, there were no errors */
  }
  adjustresults(L, nresults);
  lua_unlock(L);
  return APIstatus(status);
}
```

### 2. Bytecode Execution

Once the Lua function is called, the control is passed to the Lua virtual machine (VM) for execution. The main loop of the VM is defined in `lvm.c`:

```c
LUA_API void luaD_call (lua_State *L, StkId func, int nresults) {
  lua_lock(L);
  luaD_checkstackaux(L, 1);  /* ensure enough stack space */
  luaD_rawrunprotected(L, luaD_callprotected, func, nresults);
  lua_unlock(L);
}

LUA_API int luaD_pcall (lua_State *L, lua_CFunction func, CallInfo *c, StkId funcbase, StkId errfunc) {
  lua_lock(L);
  luaD_checkstackaux(L, 1);  /* ensure enough stack space */
  luaD_rawrunprotected(L, luaD_pcallprotected, func, c, funcbase, errfunc);
  lua_unlock(L);
}
```

The `luaD_call` and `luaD_pcall` functions prepare the stack and call the `luaD_rawrunprotected` function, which is the main entry point for the VM:
