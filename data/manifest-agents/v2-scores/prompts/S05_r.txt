# Governance Binding, Delivery-Path Sensitivity, and Targeted Patch Validation

## Claim

Four pre-committed experiments run 2026-04-19 under the handler three-gate template produce one composite claim in four parts. A fifth observation on deliberative scope-correction on a survey-class corpus is recorded as a methodology note downstream of the handler methodology's Finding 3, not as a standalone finding here.

1. **Governance binds.** A loaded CLAUDE.md raises rubric score from 3.0/10 to 8.2/10 against the same prompt (Welch t=7.84). The items that shift are the ones the prompt does not already compel; items the prompt compels fire with or without governance loaded.

2. **Delivery path is a secondary multiplier, concentrated on identity-level claims.** Identical governance content delivered inline in the user prompt scores 6.80/10 vs 7.80/10 delivered through native CLAUDE.md loading (Welch t=2.36). The gap concentrates on items requiring the agent to identify with a namespace or authority role (R2 subtract.ing namespace: 80% native → 0% inline; R10 agent/human boundary: 100% → 60%), not on items requiring instruction-following (R1, R5–R9 survive inline delivery intact).

3. **Targeted governance patches measurably close named gaps.** R3 (signer identity) and R4 (llms.txt manifest) scored 0% across four consecutive pre-patch arms spanning two experiments. A CLAUDE.md patch adding signer identity, manifest filename, format, and the edit→hash→sign loop to the Keymaster section was applied between exp 2 and exp 4. Both items scored 100% in both arms of exp 4. Two replications of 0%; two replications of 100%; patch applied between them. The patch is the actionable finding: governance reaches the manifest layer when and only when the manifest layer is explicitly named in governance.

4. **Tool surface does not mutate plan-only outputs when the prompt prohibits execution.** Full-surface and zero-surface arms produce indistinguishable rubric scores (8.40/10 vs 8.60/10, Welch t=−0.58). Both arms honored "Do not execute. Output a plan only." The R3/R4 closure post-patch was replicated in both tool-surface arms, ruling out the candidate mechanism that tool access fills the manifest-layer gap via live verification.

## Measurement

Five experiments, five pre-committed specifications. Experiments 1, 2, and 4 use the same task prompt (plan-only publish-and-verify task for subtract.ing) and the same ten-item binary rubric (R1–R10, pre-committed in the exp 1 spec) to enable cross-experiment comparison.
