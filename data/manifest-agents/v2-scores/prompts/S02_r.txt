#!/usr/bin/env bash
# hosuni — tool-use daemon for local inference
# listens on a port via nc. receives JSON, checks lookdown,
# routes inference through parsimony (coproc via named pipes),
# parses tool calls, executes, loops, logs. pure bash + jq.
# falls back to curl if parsimony pipes not present.
#
# run:   hosuni.sh [port]
# call:  echo '{"input":"list files","channel":"terminal"}' | nc localhost 8091

set -euo pipefail

export PATH="$HOME/.subtract/bin:$HOME/.subtract:$PATH"

SUBTRACT_DIR="${SUBTRACT_DIR:-$HOME/.subtract}"
HOSUNI_DIR="${HOSUNI_DIR:-$HOME/.hosuni}"
INFERENCE_HOST=$(cat "$SUBTRACT_DIR/inference_host" 2>/dev/null || echo "localhost")
INFERENCE_PORT=$(cat "$SUBTRACT_DIR/inference_port" 2>/dev/null || echo "8085")
RESEARCH_PORT=$(cat "$SUBTRACT_DIR/research_port" 2>/dev/null || echo "")
RESEARCH_HOST=$(cat "$SUBTRACT_DIR/research_host" 2>/dev/null || echo "")

PARSIMONY_OUT="$SUBTRACT_DIR/.parsimony.out"
PARSIMONY_LOCK="$SUBTRACT_DIR/.parsimony.lock"

LOOKDOWN_UNIVERSAL="$SUBTRACT_DIR/lookdown.universal.tsv"
TOOLS_FILE="$HOSUNI_DIR/tools.tsv"
LOG_DIR="$SUBTRACT_DIR/log"
DAEMON_LOG="$HOME/human/sessions/hosuni.log"

## What it uses from subtract.ing

  lookdown.tsv        The same pattern-matching routing tables that the
                      shell translation layer uses. hosuni.c loads both
                      the per-host (~/.subtract/lookdown.<hostname>.tsv)
                      and universal (~/.subtract/lookdown.universal.tsv)
                      tables at startup. In READ mode, lookdown intercepts
                      queries that match a pattern and answers instantly
                      without inference. In RUN mode, lookdown provides
                      the program list.

  inference endpoint  hosuni.c connects to whatever answers on the host
                      and port read from ~/.subtract/inference_host and
                      ~/.subtract/inference_port (default 127.0.0.1:8087).
                      In the subtract.ing formation, that's parsimony
                      proxying to llama.cpp, but hosuni.c is endpoint-
                      agnostic — anything speaking OpenAI-compatible chat
                      completions works.

  hosuni-prefetch     Before sending a query to inference, hosuni.c forks
                      ~/.subtract/bin/hosuni-prefetch, which enriches the
                      query with external context. The enriched prompt goes
                      to the model; the user's original query stays clean.

  SOUL                System prompt loaded via --soul flag. Plain text file
                      that sets the model's disposition for the conversation.

  ~/.subtract/        Config files: inference_host, inference_port.
                      Session logs written to ~/.subtract/sessions/.
                      Starred entries saved to ~/.subtract/.starred.

### Chunk 10 (from variance-lab.txt)
`$HOME/subtract.ing/SOUL.txt`. ESC clears history.

**Why:** cal.c sent single stateless prompts with no system
context. Every query was independent — no SOUL (personality/
constraints), no memory of prior turns. The model couldn't
maintain a conversation or follow standing instructions.

**Architecture decision:** cal.c builds its own messages array
and sends directly to llama-server via HTTP, rather than
routing through hosuni.sh. This avoids modifying hosuni's
non-streaming inference path and keeps the streaming socket
approach that already works. Hosuni integration for tool use
and cloud fallback deferred to a future leg.

**Rubric:** seven binary criteria pre-committed before dispatch
(messages format, SOUL loading, history ring, history lifecycle,
non-regression, JSON escaping, buffer safety).

**Dispatch:** two reviewers (local llama excluded — context
window too small for 1600-line file).
  - Sonnet 4.6 (rousseau, `claude -p`)
  - Opus 4.6 (emile, `ssh m2mini "claude -p"`)

**Results:**

| Criterion | Sonnet 4.6 | Opus 4.6 |
|-----------|-----------|----------|
| C1 messages format | pass | pass |
| C2 SOUL loading | FAIL (false positive*) | pass |
| C3 history ring | pass | pass |
| C4 history lifecycle | pass | pass |
| C5 non-regression | pass | pass |
| C6 JSON escaping | pass | pass |
| C7 buffer safety | FAIL | FAIL |

*Sonnet read the null-terminator initializer `app.soul[0] =
'\0'` as a space character and flagged it as a bug. Verified
on disk: the code is correct. First false positive from a
Claude model in this series — prior false positives were only
from the local llama.
