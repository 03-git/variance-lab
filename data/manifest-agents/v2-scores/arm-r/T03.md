The following .tsv files are in the repository, along with their formats:

1. **lookdown.universal.tsv**
   - Format: Tab-separated values.
   - Description: Contains entries that are intended to be QLoRA-portable and helpful to every household.

2. **lookdown.<hostname>.tsv**
   - Format: Tab-separated values.
   - Description: Host-specific overrides that are checked before `lookdown.universal.tsv`. These entries are used for machine-specific commands, personal shortcuts, and child-specific content.

3. **lookdown.<user>.tsv**
   - Format: Tab-separated values.
   - Description: Contains that user's confessions and taste.

4. **manifest-walk.sh**
   - Format: Outputs a TSV manifest.
   - Description: The script outputs the path, SHA256 hash, number of lines, and a description for each file in the repository that matches certain criteria (e.g., `.txt`, `.html`, `.md`, `.c`, `.h`, `.sh`, `.tsv`, `.ts`, `.conf`).

5. **lookdown.hodori.tsv**
   - Format: Tab-separated values.
   - Description: Contains confessions, not portable knowledge.

6. **session-log.spec.txt**
   - Format: Not a .tsv file, but referenced in the context.
   - Description: Specifies the canonical log format for session logs.

7. **variance-lab.txt**
   - Format: Not a .tsv file, but referenced in the context.
   - Description: Contains details about the variance lab review process and various entries and files.

8. **variance-lab-handler-methodology.txt**
   - Format: Not a .tsv file, but referenced in the context.
   - Description: Contains instructions for modifying `handler.sh` to include personal overlays and log writing.

9. **hosuni.txt**
   - Format: Not a .tsv file, but referenced in the context.
   - Description: Describes what the `hosuni` tool uses from `subtract.ing`, including routing tables and inference endpoints.

10. **llms.txt**
    - Format: Not a .tsv file, but referenced in the context.
    - Description: Lists SHA256 hashes for various files in the repository.
