The `runtime/hosuni/` directory contains several files, each serving a specific purpose:

1. **hosuni.c**:
   - This is the main source file for the application. It includes the main application logic, including handling conversation history, message building, and inference. It also talks directly to the display hardware, implementing the rendering code and font handling.

2. **hosuni.h**:
   - This file defines the platform contract and includes various types, enums, and functions for handling surfaces, events, fonts, and more. It also includes definitions for colors, font structures, and gap buffer operations. The file is crucial for defining the interface between the application and the platform-specific implementations.

3. **platform_linux.c**:
   - This file provides the implementation for the Linux platform, specifically for DRM/KMS and evdev. It includes functions for initializing the platform, handling events, and rendering.

4. **platform_macos.m**:
   - This file provides the implementation for the macOS platform, using Cocoa and NSEvent. It includes functions for initializing the platform, handling events, and rendering.

5. **stb_truetype.h**:
   - This file is a single-header library for rendering TrueType fonts. It is vendored and used for font rendering in the application.

6. **hosuni-discord.ts**:
   - This TypeScript file is part of the Discord integration. It handles polling Discord channels, sending messages, and interacting with the Discord WebSocket API. It includes functions for handling messages, sending replies, and managing the Discord state.

7. **hosuni-discord.sh**:
   - This shell script is also part of the Discord integration. It wraps the inference path for delivery over Discord, handling the interaction with the Discord API and managing the state of the Discord poller.

8. **hosuni.sh**:
   - This shell script is used for handling JSON requests and writing JSON responses. It is designed to be used in both stdin/stdout mode and when spawned by `hosuni-discord.ts` per message.

These files collectively form the core of the `hosuni` application, providing the necessary components for rendering, handling user input, performing inference, and integrating with Discord.
