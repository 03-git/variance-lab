`kiosk.sh` is a script designed to set up a display node running a specific HTML file (`multiplying.html`) via the `cage` and `cog` services. It performs several key configurations:

1. **User Setup**: It ensures the specified user (or `media` by default) has the necessary permissions and services enabled. This includes enabling lingering for the user so that user services start at boot, and ensuring the user's `.config/systemd` directory is owned by the user.

2. **Service Management**: It reloads and enables the `subtract-httpd` and `subtract-bridge` services for the specified user. These services are likely related to HTTP and bridge functionalities, respectively.

3. **Display Configuration**: It sets up the display to run `multiplying.html` on the terminal, with HTTP services running on ports 8070 and 8889.

4. **Systemd Configuration**: It configures autologin for the `tty1` terminal to ensure the specified user logs in automatically.

5. **Subtract Runtime Installation**: It installs the Subtract runtime as the user, which involves sourcing the `install.sh` script in the user's home directory.

Here is a relevant excerpt from `kiosk.sh`:

```bash
su -c "systemctl --user daemon-reload && systemctl --user enable --now subtract-httpd subtract-bridge" "$USER"

echo ""
echo "kiosk ready."
```
