To trace the delegation architecture from `governance.conf.universal` through to the runtime scripts and identify which files implement each governance reflex, we need to follow the specified reflexes and see how they are implemented in the codebase.

### Reflexes in `governance.conf.universal.txt`

1. **Library to Kernel**
   - **Description**: Name the kernel primitive before proposing any library or tool.
   - **Implementation**: This reflex is implemented in the `lookdown.universal.tsv` file, which contains commands and their corresponding kernel primitives.

2. **Format to Signature**
   - **Description**: Ask if `ssh-keygen -Y sign` can verify the format before inventing one.
   - **Implementation**: This reflex is implemented in the `lookdown.universal.tsv` file, where commands are checked for their format verification capabilities.

3. **Action to Verification**
   - **Description**: If unsigned, verify with a live read before acting.
   - **Implementation**: This reflex is implemented in the `install.sh` script, which verifies the signature of the `llms.txt` file before proceeding.

4. **Source to Signing Domain**
   - **Description**: Confirm source is under the signing key's domain before treating it as authoritative.
   - **Implementation**: This reflex is implemented in the `install.sh` script, which verifies the signature of the `llms.txt` file using the `authorized_signers` file.
