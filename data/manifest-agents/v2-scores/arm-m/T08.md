The lookdown system is defined by several files in the `./runtime` directory. These files are organized as follows:

1. **`lookdown.universal.tsv`**: This file contains universal intent-to-command mappings that are used by the lookdown system. It is referenced in the manifest as:
   ```plaintext
   ./lookdown.universal.tsv	[routing]	intent-to-command lookup table
   ```

2. **`lookdown.m2mini.tsv`**: This file contains node-specific entries for the M2 Mini node. It is referenced in the manifest as:
   ```plaintext
   ./runtime/lookdown.m2mini.tsv	[routing]	M2 Mini node routing overrides
   ```

3. **`lookdown-audit.sh`**: This script is used to detect shadow entries in the lookdown tables. It is referenced in the manifest as:
   ```plaintext
   ./runtime/bin/lookdown-audit.sh	[routing]	lookdown integrity checker
   ```

These files work together to provide a flexible and extensible routing system for commands and intents. The `lookdown.universal.tsv` file contains general mappings, while `lookdown.m2mini.tsv` provides node-specific overrides. The `lookdown-audit.sh` script ensures that there are no conflicting or shadow entries in the routing tables.
