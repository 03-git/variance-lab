The repository contains two .tsv (tab-separated values) files, which are `lookdown.example.tsv` and `lookdown.universal.tsv`. Here is a description of the format for each:

### `lookdown.example.tsv`
- **Purpose**: This file is intended to be a host-specific overrides file that can be copied to `~/.subtract/lookdown.<hostname>.tsv` to provide machine-specific commands, personal shortcuts, and child-specific content.
- **Format**:
  - Lines start with a `glob_pattern<TAB>command` format, where:
    - `glob_pattern` is a pattern that matches commands or file paths.
    - `command` is the action to be taken when the pattern is matched.
  - Additional lines can include `glob_pattern<TAB>[tag]<TAB>command` to specify the output tag (e.g., `[stdout]` or `[player]`).
  - Comments are indicated by `#` at the beginning of the line.

### `lookdown.universal.tsv`
- **Purpose**: This file serves as an intent-to-command lookup table for the system, providing a way to map user intents to specific commands or actions.
- **Format**:
  - Lines follow the `glob_pattern<TAB>[tag]<TAB>command` format, where:
    - `glob_pattern` is a pattern that matches user intents.
    - `[tag]` specifies the output tag (e.g., `[stdout]` or `[player]`), with `[stdout]` being the default if not specified.
    - `command` is the action to be taken when the pattern is matched.
  - The file is structured to prioritize specific entries over general ones, with the first matching line determining the action.
  - Comments are indicated by `#` at the beginning of the line, and sections are delineated by `# === SECTION ===` headers.

Both files use tab-separated values to define the patterns, tags, and commands, ensuring that the system can interpret and execute the appropriate actions based on user input.
