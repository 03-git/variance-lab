The `runtime/hosuni/` directory contains several files, each serving a specific purpose:

1. **`platform_linux.c`**:
   - **Description**: This file is part of the Linux platform layer for the hosuni system. It handles DRM (Direct Rendering Manager) dumb buffer and evdev (event device) integration.
   - **Key Functions**:
     - `platform_init`: Initializes the DRM framebuffer and sets up input devices.
     - `unbind_fbcon`: Unbinds the framebuffer console.
     - `signal(SIGTERM, sig_restore)` and `signal(SIGINT, sig_restore)`: Sets up signal handlers to clean up resources when the program receives SIGTERM or SIGINT signals.

2. **`hosuni.h`**:
   - **Description**: This header file defines the platform contract for hosuni, including types and function prototypes for surface management, input handling, and other operations.
   - **Key Definitions**:
     - `hosuni_surface`: A structure representing a framebuffer surface.
     - `hosuni_input_type`: Enumerates types of input (key, mouse, touch).
     - `hosuni_event`: A structure representing an input event.

3. **`hosuni.c`**:
   - **Description**: This file contains the implementation of the hosuni system, which is responsible for framebuffer computing and handling user input.
   - **Key Functions**:
     - `hosuni_draw_char`: Draws a character on the framebuffer.
     - `draw_ask`: Draws a prompt for user input and handles the input.
     - `hosuni_line_height`: Calculates the height of a line of text.

4. **`hosuni.sh`**:
   - **Description**: This is a shell script that acts as a tool-use daemon for local inference. It listens for JSON input, checks routing tables, routes inference through a subprocess, and handles tool calls.
   - **Key Variables**:
     - `SUBTRACT_DIR`, `HOSUNI_DIR`, `INFERENCE_HOST`, `INFERENCE_PORT`, `RESEARCH_PORT`, `RESEARCH_HOST`, `PARSIMONY_IN`, `PARSIMONY_OUT`, `PARSIMONY_LOCK`, `LOOKDOWN_NODE`, `LOOKDOWN_UNIVERSAL`, `TOOLS_FILE`, `LOG_DIR`, `DAEMON_LOG`, `MAX_TURNS`, `TOOL_TIMEOUT`, `SOUL_FILE`, `SYSTEM_MSG`.
   - **Key Functions**:
     - `log_daemon`: Logs messages to the daemon log file.

5. **`discord.sh`**:
   - **Description**: This script polls Discord channels and pipes messages to the hosuni daemon.
   - **Key Variables**:
     - `PIDFILE`, `TOKEN`, `WHITELIST`, `LOG`, `STATE`, `API`, `POLL_INTERVAL`, `CHANNELS`.
   - **Key Functions**:
     - `log`: Logs messages to the Discord log file.
     - `is_whitelisted`: Checks if a user is whitelisted.
     - `get_last_id` and `set_last_id`: Get and set the last message ID for a channel.
     - `send_reply`: Sends a reply to a Discord channel.

6. **`discord.ts`**:
   - **Description**: This TypeScript file implements a Discord DM daemon that listens for direct messages and channel messages, and auto-replies based on whitelisting.
   - **Key Functions**:
     - `log`: Logs messages to the Discord log file.
     - `loadSet`: Loads a set of strings from a file.
     - `apiPost`: Sends a POST request to the Discord API.

These files collectively form the core of the hosuni system, handling input, output, and communication with external systems like Discord.
