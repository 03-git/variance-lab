        Wait, let's re-check the math.
        Gap is `[gap_start, gap_end)`.
        Indices before gap: `0` to `gap_start - 1`.
        Indices after gap: `gap_start` (logical) maps to `gap_end` (physical).
        So, logical `pos` maps to `pos + (gap_end - gap_start)`.
        Let's test: If `gap_start=2, gap_end=5` (gap size 3).
        Logical `pos=0` -> `0 + (5-2) = 3`? No.
        Logical `pos=0` is physical `0`.
        Logical `pos=1` is physical `1`.
        Logical `pos=2` is physical `5` (the first char after the gap).
        Formula: `physical_pos = (pos < gap_start) ? pos : pos + (gap_end - gap_start)`.
        The code uses: `g->buf[g->gap_end + (pos - g->gap_start)]`.
        If `pos = 2` and `gap_start = 2`, it returns `g->buf[5 + 0] = g->buf[5]`.
        If `pos = 1` and `gap_start = 2`, the `if` triggers: `return g->buf[1]`.
        The logic is actually correct!
